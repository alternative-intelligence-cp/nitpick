### Phase 1: Aria (The Language)

A programming language designed from first principles:
- **TBB Types**: Every value can be an error. Handle failure gracefully.
- **Six-Stream I/O**: Express intent clearly, not just stdout/stderr
- **Arena Allocators**: Memory management that matches how humans actually think
- **Generics Done Right**: Power without complexity
- **Contribution System**: Make it easy for others to help build

**Aria isn't just another language.** It's a language designed to be understandable, teachable, and usable by people who've been failed by existing systems.

When a kid in an underfunded school wants to learn programming, they shouldn't need a CS degree to understand memory management. When a healthcare worker wants to automate their workflow, they shouldn't need to navigate enterprise Java hell.

**Aria is the language you'd design if you gave a shit about humans.**

## The Philosophy: Contribution Over Control

Traditional open source: "Here's the code, good luck figuring it out."

Our approach:
1. **Clear Tasks**: TASKS.md with specifications anyone can follow
2. **Low Barriers**: Tier 1 tasks for first-time contributors
3. **Good Documentation**: Explain the WHY, not just the WHAT
4. **Validated System**: We eat our own dog food (ARIA-001 proved the system works)

**The goal**: Someone shows up with skills and desire to help → They can contribute meaningfully within hours → The system gets better → More people can contribute → The flywheel spins.

When I'm gone, the contribution system keeps running. The documentation persists. The vision lives on.
