# The `flt128` Type (Quadruple Precision Floating Point)

**Category**: Types → Floating Point → Extended Precision  
**Syntax**: `flt128` (alias: `f128`)  
**Purpose**: 128-bit quadruple-precision floating point for extreme scientific accuracy  
**First Available**: Aria v0.0.1  
**Status**: ✅ Fully implemented (software emulation)

---

## Overview

`flt128` is Aria's **quadruple-precision floating-point type**, providing approximately **34 decimal digits** of precision with a massive range (±10⁴⁹³²). This is DOUBLE the precision of `double` (15 digits) and FIVE TIMES the precision of `float` (7 digits).

**Key Characteristics**:
- **128 bits** (16 bytes): 1 sign + 15 exponent + 112 mantissa bits
- **~34 decimal digits** of precision
- **Range**: ±1.2 × 10⁴⁹³² (cosmological scale!)
- **Deterministic**: Bit-exact results across platforms
- **Software emulation**: No native CPU hardware (very slow!)
- **~50-100× slower** than `double`

**Alias**: `f128`

**When to use**:
- ✅ Extreme precision scientific computing (quantum mechanics, astrophysics)
- ✅ Long-running simulations where error accumulation matters
- ✅ Numerical stability requirements beyond double
- ✅ Bridging gap between double and arbitrary precision

**When NOT to use**:
- ❌ Real-time applications (too slow)
- ❌ Financial calculations (use `fix256` for EXACT decimals)
- ❌ General purpose (use `double` - 15 digits sufficient)
- ❌ Memory-constrained systems (16 bytes per value)
- ❌ SIMD/GPU (no hardware support)

## Arithmetic

```aria
flt128:a = 100.0f128;
flt128:b = 3.0f128;

flt128:sum = a + b;       // 103.0 (precise to 34 digits)
flt128:diff = a - b;      // 97.0
flt128:product = a * b;   // 300.0
flt128:quotient = a / b;  // 33.333333... (34 digits)
flt128:remainder = a % b; // 1.0

// Division by zero
flt128:inf = 10.0f128 / 0.0f128;  // +INF
flt128:nan = 0.0f128 / 0.0f128;   // NaN
```

**Performance**: All operations ~50-100× slower than `double` (software emulation).

## Determinism

**Aria's flt128 is fully deterministic** (unlike IEEE 754 float/double):

```aria
// GUARANTEED bit-exact across ALL platforms
flt128:a = 0.1f128;
flt128:b = 0.2f128;
flt128:c = a + b;  // EXACTLY 0.3f128 everywhere

// Compare IEEE double (may vary)
double:x = 0.1;
double:y = 0.2;
double:z = x + y;  // Platform-dependent!
```

**Platforms**: x86_64, ARM64, RISC-V, all OS, all compilers → identical results.

## Comparisons

⚠️ **DANGER**: Direct equality is still risky (even with 34 digits)!

```aria
// ANTI-PATTERN
flt128:a = 1.0f128 / 3.0f128;
flt128:b = a * 3.0f128;
if (b == 1.0f128) {  // Might be FALSE!
}

// ✅ CORRECT: Epsilon comparison
const flt128:TOL = 1.0e-30f128;

func:almost_equal = bool(flt128:a, flt128:b, flt128:eps) {
    flt128:diff = a - b;
    if (diff < 0.0f128) { diff = -diff; }
    pass(diff < eps);
};

if (almost_equal(b, 1.0f128, TOL)) {
    stdout.write("Close enough!\n");
}
```

## Type Conversions

### Widening (Safe)

```aria
// float → flt128 (exact)
float:small = 3.14;
flt128:wide = flt128(small);

// double → flt128 (exact)
double:medium = 2.718281828459045;
flt128:wider = flt128(medium);
```

### Narrowing (Precision Loss!)

```aria
// flt128 → double (LOSES precision beyond ~15 digits)
flt128:precise = 3.14159265358979323846264338327950288f128;
double:rounded = double(precise);  // ~3.141592653589793

// Range check before narrowing
flt128:huge = 1.0e400f128;
if (huge > flt128(DBL_MAX)) {
    stderr.write("Too large for double!\n");
}
```

## Anti-Patterns

### ❌ Default Use

```aria
// WRONG: Overkill
flt128:counter = 0.0f128;  // ❌ use int32!
flt128:percent = 0.5f128;  // ❌ use float!

// CORRECT: Match precision to need
int32:counter = 0i32;
float:percent = 0.5;
```

### ❌ Financial Math

```aria
// WRONG: Still not exact!
flt128:price = 19.99f128;
flt128:total = price * 3.0f128;  // ❌ 59.97000...

// CORRECT: Exact decimals
fix256:price = fix256("19.99");
fix256:total = price * fix256("3");  // Exactly 59.97
```

### ❌ Hot Loops

```aria
// WRONG: Billion ops at 100× slowdown!
for (tbb32:i = 0i32; i < 1000000i32; ++i) {
    for (tbb32:j = 0i32; j < 1000i32; ++j) {
        flt128:r = flt128(i) * flt128(j);  // ❌ TOO SLOW
    }
}

// CORRECT: Use double, promote to flt128 at end
for (...) {
    double:r = double(i) * double(j);
    accumulator += r;
}
flt128:final = flt128(accumulator);
```

### Safe Narrowing

```aria
func:safe_to_double = Result<double>(flt128:value) {
    if (value > flt128(DBL_MAX)) { fail("Overflow"); }
    if (value < flt128(-DBL_MAX)) { fail("Underflow"); }
    pass(double(value));
};
```

### Kahan Summation

```aria
func:kahan_sum = flt128(flt128[]:values, tbb32:count) {
    flt128:sum = 0.0f128;
    flt128:c = 0.0f128;  // Compensation
    
    for (tbb32:i = 0i32; i < count; ++i) {
        flt128:y = values[i] - c;
        flt128:t = sum + y;
        c = (t - sum) - y;  // Capture lost bits
        sum = t;
    }
    pass(sum);
};
```
