# P0.2: Global Array Declarations - COMPLETE ✅

**Date**: 2026-02-17  
**Status**: FIXED - Fully Functional  
**Effort**: 1 hour actual (estimated 6-10)

## Summary

Fixed parser to accept array declarations at module/global scope. Arrays of custom types (structs, generics) were being misinterpreted as expressions instead of type annotations.

## What Was Fixed

### Root Cause
Parser's `parseStatement()` function checked if custom types were followed by `:` to identify variable declarations, but didn't account for array brackets `[...]` in between.

**Example**:
```aria
WatchPoint[16]:_watches;  // ❌ Parser saw WatchPoint[16] as expression
int32[10]:numbers;        // ✅ Worked (primitive types had array handling)
```

### Implementation

**File**: `src/frontend/parser/parser.cpp`  
**Function**: `Parser::parseStatement()`  
**Lines**: 1782-1811

Added array bracket handling after custom type identifier detection:

```cpp
// Check for custom type reference (variable declaration): CustomType:name = ...
if (peek().type == TokenType::TOKEN_IDENTIFIER) {
    size_t saved = current;
    advance(); // consume identifier
    
    // Check for generic type arguments: <...>
    if (check(TokenType::TOKEN_LESS)) {
        // ... skip generic args ...
    }
    
    // ✅ NEW: Check for array type suffix: CustomType[] or CustomType[size]
    if (check(TokenType::TOKEN_LEFT_BRACKET)) {
        advance(); // consume '['
        
        // Skip size expression if present
        if (!check(TokenType::TOKEN_RIGHT_BRACKET)) {
            int bracketDepth = 1;
            while (bracketDepth > 0 && !isAtEnd()) {
                if (check(TokenType::TOKEN_LEFT_BRACKET)) bracketDepth++;
                if (check(TokenType::TOKEN_RIGHT_BRACKET)) bracketDepth--;
                if (bracketDepth > 0) advance();
            }
        }
        
        if (check(TokenType::TOKEN_RIGHT_BRACKET)) {
            advance(); // consume ']'
        }
    }
    
    // Now check for colon (variable declaration)
    if (check(TokenType::TOKEN_COLON)) {
        current = saved; // reset position
        return parseVarDecl();  // ✅ Correctly route to variable declaration
    }
    // ...
}
```

This mirrors the existing logic for primitive type keywords (which already worked).

- ✅ Global arrays of struct types: `MyStruct[N]:name`
- ✅ Global arrays of generic types: `Option<T>[N]:name`

- ✅ Nested member access: `global_points[i].field`

## Impact

### Eliminates Workaround

**Before** (from DBUG stdlib):
```aria
wild WatchPoint->:_watches;  // Heap-allocated pointer workaround

func:init = void() {
    _watches = alloc_array<WatchPoint>(16);  // Manual allocation
}
```

**After** (idiomatic):
```aria
WatchPoint[16]:_watches;  // Stack-allocated fixed array

func:init = void() {
    _watches[0].active = 1;  // Direct access, no allocation
}
```

### Benefits
- ✅ No manual allocation/deallocation
- ✅ Better cache locality (stack vs heap)
- ✅ No pointer indirection overhead
- ✅ Cleaner, more idiomatic code
- ✅ Type-safe array bounds
