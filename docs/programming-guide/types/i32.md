# i32: 32-bit Signed Integer

## Overview

**Purpose**: General-purpose signed integer arithmetic for most numeric work.

**Characteristics**:
- 32-bit two's complement signed integer
- Range: -2,147,483,648 to +2,147,483,647 (approximately ±2.1 billion)
- Wrapping overflow (defined behavior, not UB)
- Zero overhead (native CPU instructions)
- Default integer type for Aria
- Asymmetric range (one more negative value than positive)
- Full name: `int32`, common alias: `i32`

**When to Use**:
- General counting and arithmetic (±2.1B sufficient for 99% of cases)
- Loop indices and counters
- Array sizes and indices
- Database primary keys (up to 2.1 billion records)
- Small file sizes (up to 2GB)
- Business logic calculations
- C interoperability (`int32_t`)

**When NOT to Use**:
- Large file sizes (use `i64` for files > 2GB)
- Modern timestamps (use `i64` nanoseconds - `i32` causes Year 2038 problem!)
- Safety-critical overflow detection (use `tbb32` for error propagation)
- High-precision timestamps (use `i64` for nanoseconds)
- Always-positive counts (consider `u32` for unsigned semantics)
- Tiny values with memory constraints (use `i8` or `i16`)

## i32 vs Other Integer Sizes

### Size Comparison

```aria
// 8-bit: -128 to +127
i8:tiny = 100;

// 16-bit: -32,768 to +32,767  
i16:small = 30000;

// 32-bit: -2.1 billion to +2.1 billion (DEFAULT)
i32:medium = 2000000000;

// 64-bit: -9.2 quintillion to +9.2 quintillion
i64:large = 9000000000000000000;
```

### When to Use Each Size

**i8**: Single bytes, ASCII characters, tiny enums (-128 to 127)
- Memory-critical arrays
- Network protocol bytes
- Color components (0-255 as unsigned)

**i16**: Audio samples, small coordinates (-32K to 32K)  
- 16-bit audio PCM
- Small game world coordinates
- Limited range counters

**i32**: Default for most integer work (±2.1 billion)
- **Loop indices** (most loops don't exceed 2.1B iterations)
- **Database IDs** (2.1 billion records)
- **General arithmetic** (99% of cases fit in ±2.1B)
- **C interop** (matches `int32_t`, `int` on most platforms)

**i64**: Large values, modern timestamps, big files (±9.2 quintillion)
- **Nanosecond timestamps** (good until year 2262)
- **Large file sizes** (beyond 2GB)
- **Memory addresses** (64-bit pointers)
- **Financial calculations** (high-value transactions)

## i32 vs u32: Signed vs Unsigned

### The Critical Distinction

**i32**: Signed integer (can be negative)
- Purpose: General arithmetic with positive AND negative values
- Range: -2,147,483,648 to +2,147,483,647
- Intent: "This can be negative" (temperatures, offsets, deltas)

**u32**: Unsigned integer (always non-negative)
- Purpose: Quantities that cannot be negative
- Range: 0 to 4,294,967,295 (0 to ~4.3 billion)
- Intent: "This is always positive" (sizes, counts, IDs)

### Examples of the Difference

```aria
// Signed: Temperature can be negative
i32:temperature_celsius = -40;  // -40°C (valid)

// Unsigned: Array length cannot be negative
u32:array_length = 1000000;  // 1 million elements (always positive)

// Signed: Coordinate offset (can go left/down)
i32:delta_x = -500;
i32:delta_y = 300;

// Unsigned: Memory address offset (always forward)
u32:offset = 0x1000;
```

### Conversion Pitfalls

```aria
// DANGER: Negative i32 → u32 wraps to huge positive!
i32:negative = -1;
u32:unsigned = u32(negative);  // 4,294,967,295 (wrapped!)

// Safe: Check before converting
i32:value = -100;
if (value < 0) {
    log.error("Cannot convert negative to unsigned\n");
} else {
    u32:safe = u32(value);  // Only convert if positive
}
```

## i32 vs tbb32: Wrapping vs Error Detection

### Critical Safety Difference

**i32**: Wrapping overflow (silent, continues)
- Purpose: Performance-critical arithmetic where overflow is acceptable
- Overflow: MAX + 1 = MIN (wraps around silently)
- Division by zero: Panics (runtime error)
- Range: -2,147,483,648 to +2,147,483,647 (asymmetric)

**tbb32**: Error-propagating overflow (sticky ERR sentinel)
- Purpose: Safety-critical arithmetic where overflow is catastrophic
- Overflow: MAX + 1 = ERR (error sentinel, propagates through calculations)
- Division by zero: Returns ERR (no panic)
- Range: -2,147,483,647 to +2,147,483,647 (symmetric, uses -2,147,483,648 for ERR)

### Year 2038 Problem

```aria
// i32 timestamp - SILENT WRAPAROUND (disaster!)
i32:timestamp = 2147483647;  // January 19, 2038, 03:14:07 UTC (MAX)
timestamp += 1;
// Result: -2147483648 (December 13, 1901!) 
// ⚠️ SILENT WRAPAROUND - system thinks it's 1901!

// tbb32 timestamp - DETECTS ERROR
tbb32:safe_timestamp = 2147483647;
safe_timestamp += 1;
// Result: ERR (overflow detected, can HALT system)

if (safe_timestamp == ERR) {
    log.critical("YEAR 2038 OVERFLOW DETECTED!\n");
    panic("Cannot continue with overflowed timestamp");
}
```

### When to Use Which

**Use i32 when**:
- Performance critical (zero overhead)
- Overflow mathematically impossible (bounded input)
- Wrapping is intentional (hashing, cryptography)
- C interoperability required
- Large arrays (memory efficiency)

**Use tbb32 when**:
- Safety-critical (healthcare, finance, aviation)
- **Year 2038 concerns** (32-bit timestamps)
- Error propagation needed (sticky errors)
- Overflow is catastrophic (cannot continue)
- Symmetric range needed (`abs()` must always work)

## Two's Complement Representation

### What is Two's Complement?

Two's complement is how signed integers represent negative numbers in binary.

**Positive numbers**: Normal binary (MSB = 0)
```
 0 = 0x00000000 = 0000...0000
 1 = 0x00000001 = 0000...0001
42 = 0x0000002A = 0000...00101010
```

**Negative numbers**: Inverted bits + 1 (MSB = 1)
```
 -1 = 0xFFFFFFFF = 1111...1111
-42 = 0xFFFFFFD6 = 1111...11010110
```

### Why Asymmetric Range?

```aria
// i32 range is NOT symmetric!
i32:max = 2147483647;   // 0x7FFFFFFF (01111...1111)
i32:min = -2147483648;  // 0x80000000 (10000...0000)

// One more negative value than positive!
// |min| = 2,147,483,648 (no positive representation)
```

**Why?**: Zero takes one "positive" slot (0x00000000), leaving one extra negative value.

### abs() Trap

```aria
i32:min_value = -2147483648;
// i32:result = abs(min_value);  // PANICS! Result is 2,147,483,648 (doesn't fit in i32)

// Safe alternative: Use i64 for result
i64:safe_result = i64:abs(i64(min_value));  // 2,147,483,648 (fits in i64)
```

## Literals and Type Inference

### Integer Literals

```aria
// Decimal
i32:count = 42;

// Hexadecimal (0x prefix)
i32:flags = 0xFF00;      // 65280
i32:color = 0xAABBCCDD;

// Binary (0b prefix)
i32:mask = 0b1010_1010;  // 170
i32:bits = 0b11111111;   // 255

// Underscores for readability (ignored)
i32:million = 1_000_000;
i32:billion = 1_000_000_000;
```

### Type Suffix

```aria
// Explicit i32 suffix
i32:value = 42i32;

// Useful for disambiguation
i64:large = 42i64;  // i64, not i32
u32:unsigned = 42u32;  // u32, not i32
```

### Default Inference

```aria
// Literals without suffix default to i32 in numeric contexts
count: i32 = 100;  // 100 inferred as i32

// But be careful in mixed contexts!
i64:result = 1000000000 * 2;  // Overflow! 1000000000 is i32, multiplied as i32, THEN converted
i64:correct = 1000000000i64 * 2i64;  // Safe: Both are i64, no overflow
```

## Arithmetic Operations

### Basic Arithmetic

```aria
i32:a = 100;
i32:b = 50;

i32:sum = a + b;       // 150
i32:difference = a - b; // 50
i32:product = a * b;    // 5000
i32:quotient = a / b;   // 2 (integer division, truncates)
i32:remainder = a % b;  // 0 (modulo)
```

### Integer Division (Truncates Toward Zero)

```aria
i32:a = 10;
i32:b = 3;

i32:result = a / b;  // 3 (not 3.333...)

// Negative division
i32:neg_a = -10;
i32:result2 = neg_a / b;  // -3 (truncates toward zero, not floor)
```

### Modulo with Negative Numbers

```aria
i32:a = 10;
i32:b = 3;
i32:result = a % b;  // 1

i32:neg_a = -10;
i32:result2 = neg_a % b;  // -1 (sign matches dividend)

i32:neg_b = 3;
i32:result3 = neg_a % neg_b;  // -1 (sign matches dividend)
```

### Overflow Wrapping

```aria
// Addition overflow
i32:max = 2147483647;
i32:result = max + 1;  // -2147483648 (wraps to MIN)

// Subtraction underflow
i32:min = -2147483648;
i32:result2 = min - 1;  // 2147483647 (wraps to MAX)

// Multiplication overflow
i32:large = 100000;
i32:result3 = large * large;  // Overflow! (wraps)
```

### Overflow Detection

```aria
// Manual overflow check for addition
func safe_add(a: i32, b: i32): Result<i32> {
    // Check if addition would overflow
    if (b > 0 && a > i32:MAX - b) {
        return Err("Overflow");
    }
    if (b < 0 && a < i32:MIN - b) {
        return Err("Underflow");
    }
    return Ok(a + b);
}

// Usage
Result<i32>:result = safe_add(2000000000, 200000000);
if (result.is_ok()) {
    i32:sum = result.unwrap();
} else {
    log.error("Addition overflow!\n");
}
```

```aria
i32:a = 42;
i32:b = 42;
i32:c = 100;

bool:equal = (a == b);      // true

```

## Bitwise Operations

### Bitwise Logical Operations

```aria
i32:a = 0b1100;  // 12
i32:b = 0b1010;  // 10

i32:and_result = a & b;   // 0b1000 = 8  (both bits set)
i32:or_result = a | b;    // 0b1110 = 14 (either bit set)
i32:xor_result = a ^ b;   // 0b0110 = 6  (exactly one bit set)
i32:not_result = ~a;      // 0b...11110011 = -13 (all bits flipped)
```

### Bit Shifts

```aria
i32:value = 8;  // 0b1000

// Left shift (multiply by 2^n)
i32:left = value << 2;   // 32 (0b100000)

// Logical right shift (unsigned, zero-fill)
i32:right_logical = value >>> 1;  // 4 (0b100)

// Arithmetic right shift (signed, sign-extend)
i32:neg_value = -8;
i32:right_arith = neg_value >> 1;  // -4 (sign bit preserved)
```

### Common Bit Patterns

```aria
// Check if bit is set
i32:flags = 0b1010;
i32:bit_index = 1;
bool:is_set = ((flags & (1 << bit_index)) != 0);  // true (bit 1 is set)

// Set a bit
i32:result = flags | (1 << 2);  // 0b1110 (set bit 2)

// Clear a bit
i32:result2 = flags & ~(1 << 1);  // 0b1000 (clear bit 1)

// Toggle a bit
i32:result3 = flags ^ (1 << 3);  // 0b0010 (toggle bit 3)
```

## Type Conversions

### Widening (Always Safe)

```aria
// i8 → i32 (safe, no data loss)
i8:small = -100;
i32:medium = i32(small);  // -100 (sign-extended)

// i16 → i32 (safe, no data loss)
i16:short_val = -30000;
i32:int_val = i32(short_val);  // -30000 (sign-extended)

// i32 → i64 (safe, no data loss)
i32:int32_val = -2000000000;
i64:int64_val = i64(int32_val);  // -2000000000 (sign-extended)
```

### Narrowing (Check Range!)

```aria
// i64 → i32 (UNSAFE - can overflow!)
i64:large = 5000000000;  // 5 billion (too big for i32)
i32:truncated = i32(large);  // Truncates! (undefined result)

// Safe narrowing with check
i64:value = 5000000000;
if (value >= -2147483648 && value <= 2147483647) {
    i32:safe = i32(value);
    // Use safe
} else {
    log.error("Value out of i32 range\n");
}

// Or use Result
func narrow_to_i32(value: i64): Result<i32> {
    if (value < -2147483648 || value > 2147483647) {
        return Err("Out of i32 range");
    }
    return Ok(i32(value));
}
```

### Float Conversions

```aria
// i32 → f32/f64 (may lose precision for large values)
i32:int_val = 123456789;
f32:float_val = f32(int_val);  // May lose precision (f32 only has ~7 significant digits)
f64:double_val = f64(int_val);  // Exact (f64 has ~15 significant digits)

// f32/f64 → i32 (truncates decimal part)
f64:pi = 3.14159;
i32:truncated = i32(pi);  // 3 (decimal part discarded)

f64:negative = -42.9;
i32:neg_truncated = i32(negative);  // -42 (truncates toward zero)
```

### Unsigned Conversions

```aria
// i32 → u32 (reinterpret bits)
i32:negative = -1;
u32:unsigned = u32(negative);  // 4294967295 (0xFFFFFFFF)

// u32 → i32 (reinterpret bits)
u32:large_unsigned = 3000000000;  // > i32:MAX
i32:signed = i32(large_unsigned);  // Negative! (bit pattern reinterpreted)
```

```aria
UserRecord:user = {
```

### Flags and Bitmasks

```aria
// Permission flags
const i32:READ = 0b001;   // 1
const i32:WRITE = 0b010;  // 2
const i32:EXECUTE = 0b100; // 4

i32:permissions = READ | WRITE;  // 3 (read + write)

// Check permission
bool:can_read = (permissions & READ) != 0;  // true
bool:can_execute = (permissions & EXECUTE) != 0;  // false

// Add permission

```

## Anti-Patterns

### ❌ Using i32 for Modern Timestamps

```aria
// WRONG: i32 timestamp (Year 2038 problem!)
i32:timestamp = get_current_time();  // Seconds since 1970
// Will overflow on January 19, 2038, 03:14:08 UTC!

// RIGHT: i64 nanoseconds (good until year 2262)
i64:timestamp_ns = get_current_time_nanoseconds();
```

### ❌ Using i32 for Boolean Logic

```aria
// WRONG: Using integer as boolean
i32:is_ready = 1;  // What does 1 mean? True? Ready? Count?

if (is_ready) {  // ERROR: i32 cannot be used as bool
    launch();
}

// RIGHT: Use bool type
bool:is_ready = true;

if (is_ready) {
    launch();
}
```

### ❌ Ignoring Overflow in Accumulation

```aria
// WRONG: Accumulating without overflow check
i32:total_bytes = 0;

for (file in files) {
    total_bytes += file.size();  // Can overflow silently!
}

// If total > 2GB, total_bytes wraps negative!

// RIGHT: Use i64 for large sums
i64:total_bytes = 0;

for (file in files) {
    total_bytes += i64(file.size());
}
```

### ❌ Narrowing Without Range Check

```aria
// WRONG: Blind narrowing
i64:large = get_large_value();
i32:small = i32(large);  // Silently truncates!

// RIGHT: Check range first
i64:large = get_large_value();
if (large >= -2147483648 && large <= 2147483647) {
    i32:small = i32(large);
    // Use small
} else {
    log.error("Value out of i32 range\n");
}
```

### ❌ Using abs() on INT32_MIN

```aria
// WRONG: abs() on minimum value
i32:value = -2147483648;
// i32:absolute = abs(value);  // PANICS! (2147483648 doesn't fit in i32)

// RIGHT: Check first or use wider type
i32:value = -2147483648;
if (value == i32:MIN) {
    log.error("Cannot take abs of i32:MIN\n");
} else {
    i32:absolute = abs(value);
}

// Or widen to i64
i64:abs_value = i64:abs(i64(value));  // 2147483648 (fits in i64)
```

## C Interoperability

### ABI Compatibility

```aria
// Aria i32 is ABI-compatible with C int32_t
extern "C" {
    func get_file_size(path: *char): i32;
    func get_process_id(): i32;
    func read_bytes(fd: i32, buffer: *byte, count: i32): i32;
}

// Direct usage (no conversion needed)
i32:size = get_file_size("data.txt");
i32:pid = get_process_id();
```

### Platform int Compatibility

```aria
// C's "int" is usually i32 on modern platforms
extern "C" {
    func printf(format: *char, ...);  // int printf(const char*, ...)
    func strlen(str: *char): i32;     // size_t strlen (size_t may differ!)
}

// Usage
i32:len = strlen("Hello");
printf("Length: %d\n", len);
```

### Passing by Value vs Pointer

```aria
// By value (copy)
extern "C" {
    func process_int(value: i32): i32;
}

i32:input = 42;
i32:output = process_int(input);

// By pointer (reference)
extern "C" {
    func modify_int(ptr: *i32);
}

i32:value = 100;
modify_int(&value);  // C function can modify value
```

## Performance Characteristics

### Zero Overhead

```aria
i32:a = 1000;
i32:b = 2000;
i32:sum = a + b;
```

**Generated assembly** (x86-64):
```asm
mov eax, 1000    ; Load 1000 into EAX register
add eax, 2000    ; Add 2000 to EAX (single instruction!)
; Result in EAX
```

**One CPU instruction** for addition, subtraction, multiplication (when in registers).

### Register Allocation

Modern CPUs have 16+ integer registers (x86-64). Frequently used i32 values stay in registers (fastest possible access).

```aria
// Hot loop - values likely stay in registers
i32:sum = 0;
for (i: i32 = 0; i < 1000000; i += 1) {
    sum += i;
}
// No memory access needed - all in registers!
```

### Memory Efficiency

```aria
// Array of 1 million i32 values
i32:data[1000000];
// Memory: 4 MB (4 bytes × 1 million)

// Compare to i64
i64:large_data[1000000];
// Memory: 8 MB (8 bytes × 1 million) - 2× larger
```

**Trade-off**: i32 uses half the memory of i64, but has smaller range.

### Cache Efficiency

Smaller data types (i32 vs i64) improve cache utilization:
- More values fit in cache line (64 bytes)
- Fewer cache misses
- Better memory bandwidth utilization

**Example**: 16 i32 values fit in one cache line, only 8 i64 values.

## Common Gotchas

### 1. Overflow Wraps Silently

```aria
i32:max = 2147483647;
i32:result = max + 1;  // -2147483648 (no error, no warning!)

// Always dangerous in production
```

### 2. Mixed-Type Arithmetic

```aria
i32:a = 1000000;
i64:b = 2000000000;

// What type is the result?
i64:result = a * 2000;  // Overflow! a*2000 computed as i32, THEN converted to i64

// Correct: Explicitly widen first
i64:correct = i64(a) * 2000i64;
```

### 3. Division by Zero Panics

```aria
i32:a = 100;
i32:b = 0;
// i32:result = a / b;  // PANIC! Runtime error

// Always check divisor
if (b != 0) {
    i32:result = a / b;
} else {
    log.error("Division by zero\n");
}
```

### 4. Negative Modulo Matches Dividend

```aria
i32:a = -10;
i32:b = 3;
i32:result = a % b;  // -1 (not 2!)

// Sign matches dividend (a), not divisor (b)
```

### 5. Integer Division Truncates

```aria
i32:a = 10;
i32:b = 3;
i32:result = a / b;  // 3 (not 3.333...)

// For exact division, use floating point
f64:exact = f64(a) / f64(b);  // 3.333...
```

## Best Practices

### ✅ DO: Use i32 as Default Integer

```aria
// For general counting, indexing, most arithmetic
i32:count = 0;
i32:index = 5;
i32:total = 1000;
```

### ✅ DO: Use i64 for Large Values

```aria
// File sizes beyond 2GB
i64:file_size = get_file_size("large_video.mp4");

// Modern timestamps (nanoseconds)
i64:timestamp_ns = get_current_time_nanoseconds();

// Large sums
i64:total_revenue_cents = calculate_yearly_revenue();
```

### ✅ DO: Check Range When Narrowing

```aria
// Always validate before narrowing
i64:large_value = get_database_id();

if (large_value >= i32:MIN && large_value <= i32:MAX) {
    i32:id = i32(large_value);
    // Safe to use
} else {
    log.error("ID out of i32 range\n");
}
```

### ✅ DO: Use tbb32 for Safety-Critical Code

```aria
// Financial calculations
tbb32:account_balance = get_balance();
tbb32:transaction = get_transaction();
tbb32:new_balance = account_balance + transaction;

if (new_balance == ERR) {
    log.critical("Balance overflow - transaction rejected\n");
    rollback();
}
```

### ❌ DON'T: Use i32 for Timestamps in New Code

```aria
// WRONG: Will break in 2038
i32:timestamp = get_current_time();  // Seconds since 1970

// RIGHT: Use i64 nanoseconds
i64:timestamp_ns = get_current_time_nanoseconds();  // Good until 2262
```

### ❌ DON'T: Assume No Overflow

```aria
// WRONG: Silent overflow
i32:sum = 0;
for (i: i32 = 0; i < 100000; i += 1) {
    sum += 50000;  // Will overflow!
}

// RIGHT: Use i64 or check
i64:sum = 0;
for (i: i32 = 0; i < 100000; i += 1) {
    sum += 50000i64;  // Safe
}
```

### ❌ DON'T: Mix Signs Carelessly

```aria
// WRONG: Mixing signed and unsigned
i32:signed_value = -100;
u32:unsigned_value = u32(signed_value);  // Wraps to 4,294,967,196!

// RIGHT: Check sign first
i32:value = get_input();
if (value < 0) {
    log.error("Expected positive value\n");
} else {
    u32:unsigned = u32(value);  // Safe
}
```
