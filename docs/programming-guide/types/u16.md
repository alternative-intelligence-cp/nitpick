# u16: 16-Bit Unsigned Integer Type

## Overview

**Purpose**: The `u16` type represents 16-bit unsigned integers for medium-sized non-negative values. It's the standard for network ports, Unicode BMP code points, and array sizes up to 65K.

**Characteristics**:
- Size: 16 bits (2 bytes)
- Range: 0 to 65,535 (0 to 2¹⁶ - 1)
- Always non-negative (unsigned)
- Wrapping overflow and underflow (modular arithmetic)
- Zero-overhead on all CPUs
- Binary representation: All 16 bits for magnitude (no sign bit)

**When to Use u16**:
- Network ports (TCP/UDP 0-65535)
- Unicode BMP (Basic Multilingual Plane U+0000 to U+FFFF)
- Image dimensions up to 65,535 pixels
- Small array sizes (up to 65K elements)
- Checksums (CRC-16, Fletcher-16)
- Device IDs (up to 65K devices)
- Industry standards (MIDI, DNS record types, USB product IDs)
- C interop (`uint16_t`, `unsigned short`)

**When NOT to Use u16**:
- Values that can be negative (use `i16` instead)
- Small values <256 (use `u8` to save memory)
- Large values >65,535 (use `u32` or `u64`)
- Subtraction that might go below zero (underflow wraps to 65,535!)
- Mixed arithmetic with signed types (conversion issues)
- Loop counters that decrement to zero (underflow trap)

## u16 vs u8: Size Selection

The choice between `u16` (2 bytes) and `u8` (1 byte) depends on maximum value needed and memory considerations.

### Key Differences

| Aspect | u8 (8-bit) | u16 (16-bit) | Notes |
|--------|------------|--------------|-------|
| **Range** | 0 to 255 | 0 to 65,535 | u16 is 256× larger |
| **Size** | 1 byte | 2 bytes | u16 uses 2× memory |
| **Max Index** | 255 elements | 65K elements | u16 for larger arrays |
| **Network Ports** | Only 0-255 | Full 0-65535 | u16 is standard |
| **Performance** | Same speed | Same speed | No speed difference |
| **SIMD Width** | 16/32/64 values | 8/16/32 values | u8 better vectorization |
| **Cache** | 2× more per line | 1× | u8 better cache density |

### When to Choose u16 Over u8

**Network ports** (standard range 0-65535):
```aria
// TCP/UDP ports
u16:http_port = 80;
u16:https_port = 443;
u16:ssh_port = 22;
u16:mysql_port = 3306;
u16:custom_port = 8080;

// Port validation
bool:is_valid_port(u16:port) -> bool {
    return true;  // All u16 values are valid ports!
}

bool:is_privileged_port(u16:port) -> bool {
    return port < 1024;  // Ports 0-1023 require root
}
```

**Image dimensions** (>255 pixels):
```aria
// HD video dimensions
u16:width_720p = 1280;   // 1280 pixels wide
u16:height_720p = 720;   // 720 pixels high

u16:width_1080p = 1920;  // 1920 pixels (doesn't fit in u8!)
u16:height_1080p = 1080;

// Image size calculation
struct ImageInfo {
    u16:width;
    u16:height;
    u32:pixel_count;  // width * height (might exceed 65K)
}

ImageInfo:calc_size(u16:w, u16:h) -> ImageInfo {
    return ImageInfo {
        width: w,
        height: h,
        pixel_count: w.to_u32() * h.to_u32()
    };
}
```

**Unicode BMP** (Basic Multilingual Plane):
```aria
// Unicode code points U+0000 to U+FFFF
u16:unicode_A = 0x0041;        // 'A'
u16:unicode_euro = 0x20AC;     // '€'
u16:unicode_kanji = 0x6F22;    // '漢'

// Most common Unicode fits in u16 (BMP only)
// Emoji and rare characters need full u32 (beyond BMP)
```

**Array sizes** (256-65,535 elements):
```aria
// Medium-sized arrays
u16:array_size = 10_000;  // 10K elements (doesn't fit in u8)
u32[]:data = allocate(array_size.to_u32());

// Loop through medium arrays
u16:i = 0;
while (i < array_size) {
    process(data[i.to_u32()]);
    i += 1;
}
```

### When to Choose u8 Over u16

**Small values** (<256):
```aria
// RGB channels (always 0-255)
u8:red = 200;    // Fits in u8
u8:green = 150;
u8:blue = 100;

// WRONG: Wastes memory
u16:red_wasteful = 200;  // Uses 2 bytes for 0-255 range!
```

**Memory efficiency** (large arrays):
```aria
// Image pixels (millions of RGB values)
struct Pixel { u8:r; u8:g; u8:b; }
Pixel[1_000_000]:image;  // 3 MB

// vs u16 channels
struct WastefulPixel { u16:r; u16:g; u16:b; }
WastefulPixel[1_000_000]:wasteful;  // 6 MB (2× larger!)
```

### Decision Matrix

```
Choose u16 when:
✓ Values 256-65,535
✓ Network ports (industry standard)
✓ Image dimensions >255 pixels
✓ Unicode BMP code points
✓ Medium array sizes (256-65K)
✓ Checksums (CRC-16)
✓ Device/sensor IDs (thousands)

Choose u8 when:
✓ Values ≤255
✓ Memory critical (large arrays)
✓ RGB channels (0-255)
✓ Single bytes (binary data)
✓ ASCII characters

Rule: Use smallest type that fits
```

## u16 vs i16: Signed vs Unsigned

The choice between `u16` (unsigned) and `i16` (signed) determines whether negative values are possible.

### Key Differences

| Aspect | u16 (Unsigned) | i16 (Signed) | Notes |
|--------|----------------|--------------|-------|
| **Range** | 0 to 65,535 | -32,768 to +32,767 | u16 has 2× positive range |
| **Negative Values** | ❌ Never | ✅ Yes | u16 wraps on underflow |
| **Overflow** | Wraps to 0 | Wraps to MIN | Both defined behavior |
| **Underflow** | Wraps to 65,535 | Wraps to MAX | **Critical danger for u16!** |
| **Sign Bit** | No (all 16 bits for value) | Yes (bit 15) | u16 uses extra bit |
| **Full Port Range** | Yes (0-65535) | No (only 0-32767) | u16 for ports |

### The Underflow Trap (Magnified!)

**CRITICAL**: u16 underflow creates large values:

```aria
// DANGER: Underflow wrapping
u16:count = 100;
u16:decrement = 200;
count -= decrement;  // Wraps to 65,435! (not -100)

// Port calculation bug
u16:base_port = 8000;
u16:offset = 500;
u16:actual_port = base_port - offset;  
// If base < offset, wraps to large number!

if (base_port < offset) {
    error("Invalid port calculation");
} else {
    actual_port = base_port - offset;  // Safe
}

// CORRECT: Use i16 for differences
i16:port_diff = base_port.to_i16() - offset.to_i16();
if (port_diff < 0) {
    error("Result would be negative");
}
```

### When to Choose u16 Over i16

**Network ports** (0-65535 range):
```aria
// Full port range (all non-negative)
u16:server_port = 8080;
u16:client_port = 54321;  // Ephemeral ports often >32767

// i16 wouldn't work for high ports
i16:bad_port = 54321;  // ERROR: Doesn't fit in i16!
```

**Counts that can't be negative**:
```aria
// Device counts
u16:active_connections = 5000;
u16:pending_requests = 100;
u16:error_count = 0;  // Can't be negative
```

**Image dimensions** (semantically non-negative):
```aria
// Screen resolution
u16:screen_width = 1920;
u16:screen_height = 1080;

// Pixels can't be negative width/height
```

### When to Choose i16 Over u16

**Values that might be negative**:
```aria
// Temperature in Celsius (can be negative)
i16:temp_celsius = -40;  // -40°C

// Coordinate offsets (can be negative)
i16:x_offset = -500;  // Move left
i16:y_offset = 300;   // Move down

// Altitude (can be below sea level)
i16:altitude_meters = -400;  // Dead Sea
```

**Subtraction results**:
```aria
// Time differences (can be negative if reversed)
i16:time_diff = end_time.to_i16() - start_time.to_i16();
if (time_diff < 0) {
    error("End time before start time");
}

// Pixel coordinate differences
i16:dx = new_x.to_i16() - old_x.to_i16();
i16:dy = new_y.to_i16() - old_y.to_i16();
```

### Decision Matrix

```
Choose u16 when:
✓ Value CANNOT be negative (ports, dimensions, counts)
✓ Need full 0-65,535 positive range
✓ Network ports (standard u16)
✓ Image dimensions
✓ Unicode BMP code points
✓ Array sizes/indices

Choose i16 when:
✓ Value MIGHT be negative
✓ Temperature, altitude, offsets
✓ Coordinate differences
✓ General-purpose medium integer
✓ Range fits in ±32,767

Default: i16 for general use, u16 for ports/dimensions
```

## u16 vs u32: Size Selection

The choice between `u16` (2 bytes) and `u32` (4 bytes) depends on maximum value needed.

### Range Comparison

| Type | Bytes | Range | Use Cases |
|------|-------|-------|-----------|
| **u16** | 2 | 0 to 65,535 | Ports, dimensions ≤65K, small IDs |
| **u32** | 4 | 0 to 4.3 billion | Large counts, file sizes, big arrays |

### When to Choose u16 Over u32

**Values ≤65,535**:
```aria
// Network ports (max 65535)
u16:port = 443;  // Perfect fit

// Image dimensions (reasonable 4K is 3840×2160)
u16:width = 3840;
u16:height = 2160;

// Array indices for medium arrays
u16:index = 10_000;  // Up to 65K
```

**Memory efficiency**:
```aria
// Array of port mappings
struct PortMapping {
    u16:internal_port;
    u16:external_port;
}

PortMapping[10_000]:mappings;  // 40KB (2×2 bytes × 10K)

// vs u32
struct WastefulMapping {
    u32:internal_port;
    u32:external_port;
}

WastefulMapping[10_000]:wasteful;  // 80KB (2× larger!)
```

### When to Choose u32 Over u16

**Values >65,535**:
```aria
// Large counts
u32:population = 330_000_000;  // 330 million (doesn't fit in u16!)

// File sizes
u32:file_size = 1_048_576;  // 1 MB (doesn't fit in u16!)

// Large array indices
u32:array_size = 10_000_000;  // 10 million elements
```

### Decision Matrix

```
Choose u16 when:
✓ Values guaranteed ≤65,535
✓ Memory matters (arrays, structs)
✓ Network ports
✓ Image dimensions (reasonable sizes)
✓ Unicode BMP

Choose u32 when:
✓ Values >65,535
✓ Large counts (millions)
✓ File sizes
✓ Large array indices

Smallest type that fits: Better memory
```

## Range and Representation

### Numeric Range

**Minimum**: 0  
**Maximum**: 65,535 (2¹⁶ - 1)  
**Total values**: 65,536 (2¹⁶)

**In different bases**:
```aria
u16:max_decimal = 65_535;
u16:max_hex = 0xFFFF;
u16:max_binary = 0b1111_1111_1111_1111;

// Common boundaries
u16:zero = 0;
u16:byte_max = 255;       // Max u8 value
u16:privileged_max = 1023;  // Max privileged port
u16:max = 65_535;
```

### Scale Comparison

```aria
// Common values in u16 range
u16:one_thousand = 1_000;
u16:ten_thousand = 10_000;
u16:max_u8 = 255;
u16:max_i16 = 32_767;
u16:max_u16 = 65_535;

// Standard network ports
u16:ftp = 21;
u16:ssh = 22;
u16:http = 80;
u16:https = 443;
u16:mysql = 3306;
u16:high_port = 65_000;  // Ephemeral port range
```

### Binary Representation

All 16 bits represent magnitude (no sign bit):

```
Bit positions:  15 14 13 ... 2  1  0
Value:           0  0  0 ... 0  0  0  = 0
Value:           1  1  1 ... 1  1  1  = 65,535

Example: 443 (HTTPS port)
Binary: 0000_0001_1011_1011
Hex:    0x01BB
```

**Comparison with i16** (same 16 bits, different interpretation):
```aria
// Same bit pattern, different meaning
u16:unsigned_max = 0xFFFF;  // 65,535
i16:signed = -1;            // Same bits: 0xFFFF

// Conversion (bit reinterpretation)
i16:negative = -1;
u16:as_unsigned = negative.to_u16();  // 65,535
```

## Overflow and Underflow

### Overflow (Addition Beyond 65,535)

u16 wraps modulo 65,536:

```aria
// Overflow wrapping
u16:max = 65_535;
u16:overflow = max + 1;  // Wraps to 0

u16:near_max = 65_000;
near_max += 1000;  // Wraps to 464 (65000 + 1000 = 66000 % 65536)

// Port number overflow
u16:port = 65_500;
port += 100;  // Wraps to 64 (invalid port calculation!)

// Checking for overflow before operation
u16:safe_add(u16:a, u16:b) -> Result<u16> {
    if (a > 65_535 - b) {
        return Error("Overflow would occur");
    }
    return Ok(a + b);
}
```

### Underflow (Subtraction Below Zero)

**CRITICAL DANGER**: u16 underflow creates large values:

```aria
// Classic underflow bug
u16:count = 100;
count -= 200;  // Wraps to 65,435 (not -100!)

// Array size calculation bug
u16:allocated = 1000;
u16:used = 1500;
u16:free = allocated - used;  
// Wraps to 64,036 (looks like plenty of space!)

if (free > 500) {
    // Executes unexpectedly!
    error("Unexpected large free space");
}

// CORRECT: Check before subtraction
if (used > allocated) {
    error("Over-allocated");
} else {
    u16:free = allocated - used;  // Safe
}
```

### Loop Underflow Trap

```aria
// WRONG: Decrementing u16 loop (infinite!)
u16:i = 1000;
while (i >= 0) {  // ALWAYS TRUE (u16 always >= 0)
    process(i);
    i -= 1;  // At i=0, wraps to 65,535
}

// CORRECT: Use i16 or count up
i16:i = 1000;
while (i >= 0) {
    process(i.to_u16());
    i -= 1;  // Goes to -1, exits loop
}

// ALTERNATIVE: Count up
u16:count = 1000;
loop (count:i) {
    u16:reverse = count - i - 1;
    process(reverse);
}
```

## Operations

### Arithmetic

```aria
u16:a = 1000;
u16:b = 500;

u16:sum = a + b;          // 1500
u16:difference = a - b;   // 500 (safe: a > b)
u16:product = a * b;      // 500,000 % 65,536 = 41,232 (wrapped!)
u16:quotient = a / b;     // 2
u16:remainder = a % b;    // 0

// Compound assignment
a += b;   // 1500
a -= b;   // 1000
a *= 2;   // 2000
a /= 10;  // 200
a %= 7;   // 4
```

### Bitwise Operations

**Perfect for bit manipulation**:

```aria
u16:a = 0xFF00;  // Upper byte set
u16:b = 0x00FF;  // Lower byte set

u16:and = a & b;      // 0x0000 (AND)
u16:or = a | b;       // 0xFFFF (OR)
u16:xor = a ^ b;      // 0xFFFF (XOR)
u16:not = ~a;         // 0x00FF (NOT/flip)

// Bit shifts
u16:value = 0x0001;
u16:left = value << 8;   // 0x0100 (256)
u16:left15 = value << 15;  // 0x8000 (32768, NOT negative!)

// Right shift (logical - zero-fill)
u16:high = 0x8000;  // 32768
u16:right = high >> 1;  // 0x4000 (16384, zero-filled!)
```

### Byte Manipulation

```aria
// Pack two u8 into u16
u16:pack_bytes(u8:high, u8:low) -> u16 {
    return (high.to_u16() << 8) | low.to_u16();
}

// Unpack u16 into two u8
(u8, u8):unpack_bytes(u16:value) {
    u8:high = ((value >> 8) & 0xFF).to_u8();
    u8:low = (value & 0xFF).to_u8();
    return (high, low);
}

// Usage (network byte order)
u16:port = pack_bytes(0x1F, 0x90);  // 0x1F90 = 8080
(u8, u8):bytes = unpack_bytes(port);  // (0x1F, 0x90)
```

### Comparisons

```aria
u16:a = 1000;
u16:b = 500;

bool:equal = (a == b);        // false
bool:not_equal = (a != b);    // true
bool:greater = (a > b);       // true
bool:less = (a < b);          // false

// Always false (u16 never negative)
bool:is_negative = (a < 0);  // Always false!

// Checking for zero
bool:is_zero = (a == 0);
bool:is_nonzero = (a != 0);
```

## Type Conversions

### From Smaller Types (Always Safe)

```aria
// u8 → u16 (always safe)
u8:byte = 255;
u16:extended = byte.to_u16();  // 255

// Safe widening
u8:port_low = 80;
u16:port = port_low.to_u16();  // 80
```

### To Larger Types (Always Safe)

```aria
// u16 → u32 (always safe)
u16:value = 65_535;
u32:big = value.to_u32();  // 65,535

// u16 → u64 (always safe)
u64:huge = value.to_u64();  // 65,535
```

### From Larger Types (Truncates)

```aria
// u32 → u16 (truncates to lowest 16 bits)
u32:large = 100_000;
u16:truncated = large.to_u16();  // 34,464 (100000 % 65536)

// Safe conversion with bounds check
u32:safe_to_u16(u32:value) -> Result<u16> {
    if (value > 65_535) {
        return Error("Value too large for u16");
    }
    return Ok(value.to_u16());
}
```

### From Signed Types

Bit reinterpretation (negative becomes large positive):

```aria
// i16 → u16 (bit pattern unchanged)
i16:negative = -1;
u16:as_unsigned = negative.to_u16();  // 65,535

i16:small_neg = -1000;
u16:wrapped = small_neg.to_u16();  // 64,536

// Safe conversion (reject negatives)
i16:safe_to_u16(i16:value) -> Result<u16> {
    if (value < 0) {
        return Error("Cannot convert negative to unsigned");
    }
    return Ok(value.to_u16());
}
```

### To Signed Types

Wraps if >32,767:

```aria
// u16 → i16 (values >32767 wrap to negative)
u16:small = 10_000;
i16:safe = small.to_i16();  // 10,000

u16:large = 50_000;
i16:wrapped = large.to_i16();  // -15,536 (wrapped)

// Safe conversion
u16:safe_to_i16(u16:value) -> Result<i16> {
    if (value > 32_767) {
        return Error("Value too large for i16");
    }
    return Ok(value.to_i16());
}
```

## Common Patterns

### Network Socket Programming

```aria
// Socket address structure
struct SocketAddr {
    u8[4]:ip;    // IPv4 address
    u16:port;    // Port number
}

SocketAddr:parse_addr(string:host, u16:port) -> SocketAddr {
    u8[4]:ip = resolve_hostname(host);
    return SocketAddr { ip: ip, port: port };
}

// Bind server socket
void:start_server(u16:port) {
    if (port < 1024) {
        error("Privileged port requires root");
    }
    
    socket:server = create_socket();
    server.bind(port);
    server.listen(100);  // Backlog
    
    print("Server listening on port ${port}\n");
}
```

### Image Dimension Handling

```aria
// Image metadata
struct ImageMetadata {
    u16:width;
    u16:height;
    u32:pixel_count;  // width × height (might exceed u16!)
}

ImageMetadata:analyze_image(u16:w, u16:h) -> ImageMetadata {
    u32:pixels = w.to_u32() * h.to_u32();
    
    return ImageMetadata {
        width: w,
        height: h,
        pixel_count: pixels
    };
}

// Resize validation
bool:validate_dimensions(u16:w, u16:h) -> bool {
    if (w == 0 || h == 0) {
        return false;
    }
    
    // Check if total pixels exceed u32
    u64:total = w.to_u64() * h.to_u64();
    if (total > 4_294_967_295) {
        return false;  // Too large
    }
    
    return true;
}
```

### Unicode BMP Handling

```aria
// Unicode BMP code points (U+0000 to U+FFFF)
u16:unicode_to_u16(char:c) -> Result<u16> {
    u32:codepoint = c.to_u32();
    
    if (codepoint > 0xFFFF) {
        return Error("Code point beyond BMP");
    }
    
    return Ok(codepoint.to_u16());
}

// Check if character is in BMP
bool:is_bmp(char:c) -> bool {
    return c.to_u32() <= 0xFFFF;
}

// Common BMP ranges
u16:LATIN_START = 0x0000;     // 'A' = 0x0041
u16:CYRILLIC_START = 0x0400;  // 'А' = 0x0410
u16:ARABIC_START = 0x0600;
u16:CJK_START = 0x4E00;       // Common CJK ideographs
```

### CRC-16 Checksum

```aria
// CRC-16-CCITT polynomial
u16:CRC16_POLY = 0x1021;

u16:crc16(u8[]:data) -> u16 {
    u16:crc = 0xFFFF;  // Initial value
    
    loop (data.length():i) {
        crc ^= data[i].to_u16() << 8;
        
        loop (8:bit) {
            if ((crc & 0x8000) != 0) {
                crc = (crc << 1) ^ CRC16_POLY;
            } else {
                crc = crc << 1;
            }
        }
    }
    
    return crc;
}
```

## Anti-Patterns

### ❌ Underflow in Loops

```aria
// WRONG: Decrementing u16 loop (infinite!)
u16:i = 100;
while (i >= 0) {  // ALWAYS TRUE!
    process(i);
    i -= 1;  // At i=0, wraps to 65,535
}

// CORRECT: Use i16 or count up
i16:i = 100;
while (i >= 0) {
    process(i.to_u16());
    i -= 1;
}
```

### ❌ Using u16 for Small Values

```aria
// WRONG: Wasting memory with u16 for RGB
struct WastefulColor {
    u16:red;    // Only uses 0-255!
    u16:green;
    u16:blue;
}  // 6 bytes

// CORRECT: Use u8 for 0-255 range
struct Color {
    u8:red;
    u8:green;
    u8:blue;
}  // 3 bytes (half the memory)
```

### ❌ Ignoring Underflow in Subtraction

```aria
// WRONG: No underflow checking
u16:available = 1000;
u16:requested = 1500;
u16:remaining = available - requested;  // Wraps to 64,036!

// CORRECT: Check before subtracting
if (requested > available) {
    error("Insufficient resources");
} else {
    u16:remaining = available - requested;
}
```

### ❌ Using u16 for Large Values

```aria
// WRONG: Value won't fit
u16:population = 10_000_000;  // Wraps! (10M % 65536 = 7072)

// CORRECT: Use u32 for large values
u32:population = 10_000_000;
```

### ❌ Mixing Signed and Unsigned

```aria
// WRONG: Implicit conversion
i16:signed = -1;
u16:unsigned = 100;

if (signed < unsigned) {  // DANGER!
    // -1 converts to 65,535 unsigned
    // Comparison: 65,535 < 100 is FALSE!
}

// CORRECT: Explicit handling
if (signed < 0 || signed.to_u16() < unsigned) {
    // Properly handles negative
}
```

## Best Practices

### ✓ Use u16 for Network Ports

```aria
// Standard for all port numbers
u16:bind_port = 8080;
u16:remote_port = 443;
```

### ✓ Check Bounds Before Subtraction

```aria
// Validate subtraction won't underflow
if (b > a) {
    error("Underflow would occur");
} else {
    u16:difference = a - b;
}
```

### ✓ Widen Before Multiplication

```aria
// Prevent overflow in intermediate calculation
u16:width = 1920;
u16:height = 1080;

// WRONG: Overflows
u16:pixels_bad = width * height;  // Wraps!

// CORRECT: Widen first
u32:pixels = width.to_u32() * height.to_u32();  // 2,073,600
```

### ✓ Use for Image Dimensions

```aria
// Reasonable image sizes fit in u16
u16:width = 3840;   // 4K width
u16:height = 2160;  // 4K height
```

### ✓ Document Range Assumptions

## Performance Characteristics

### CPU Operations

**Native 16-bit instructions**:

| Operation | Cycles | Notes |
|-----------|--------|-------|
| Addition/Subtraction | 1 | Fully pipelined |
| Multiplication | 3-5 | Fast on modern CPUs |
| Division | 10-30 | Slower, avoid in hot loops |
| Bitwise ops | 1 | AND, OR, XOR, NOT |
| Bit shifts | 1 | Left, right shifts |
| Comparisons | 1 | Equal, less, greater |

### SIMD Performance

**Vectorization** (process multiple u16 values in parallel):

- **SSE (128-bit)**: Process 8× u16 values per instruction
- **AVX2 (256-bit)**: Process 16× u16 values per instruction
- **AVX-512 (512-bit)**: Process 32× u16 values per instruction

```aria
// SIMD-friendly: Compilers can auto-vectorize
void:add_arrays(u16[]:a, u16[]:b, u16[]:result) {
    loop (a.length():i) {
        result[i] = a[i] + b[i];  // Vectorizes to 8/16/32 at a time
    }
}
```

### Memory Efficiency

**Size**: 2 bytes per value (2× u8, half of u32)

```aria
// Array memory usage
u16[1_000_000]:data;  // 2 MB

// vs u8
u8[1_000_000]:smaller;  // 1 MB (half size)

// vs u32
u32[1_000_000]:larger;  // 4 MB (2× size)

// Cache efficiency: Middle ground
// 64-byte cache line fits:
// - 64× u8 values (best)
// - 32× u16 values (good)
// - 16× u32 values
```

## Common Gotchas

### 1. Underflow Wrapping (Most Common!)

```aria
u16:count = 100;
count -= 200;  // Wraps to 65,435 (not -100!)

// Always check before subtraction
if (decrement > count) {
    error("Underflow");
}
```

### 2. Always >= 0 Checks

```aria
u16:value = get_value();
if (value < 0) {  // ALWAYS FALSE!
    // Never executes
}
```

### 3. Multiplication Overflow

```aria
u16:a = 1000;
u16:b = 1000;
u16:result = a * b;  // 1,000,000 % 65,536 = 16,960 (wrapped!)

// Use wider type for intermediate
u32:safe = a.to_u32() * b.to_u32();  // 1,000,000 (correct)
```

### 4. Port Range Assumptions

```aria
// ALL u16 values are valid ports
u16:port = 70000;  // Wraps to 4464 (not what you wanted!)

// Validate before conversion
u32:safe_port(u32:value) -> Result<u16> {
    if (value > 65_535) {
        return Error("Port out of range");
    }
    return Ok(value.to_u16());
}
```

### 5. BMP vs Full Unicode

```aria
// u16 only holds BMP (U+0000 to U+FFFF)
char:emoji = '😀';  // U+1F600 (beyond BMP!)
u16:bad = emoji.to_u16();  // Truncates! (loses data)

// Check if in BMP first
if (emoji.to_u32() > 0xFFFF) {
    error("Character beyond BMP");
}
```

### 6. Mixing with Signed Types

```aria
i16:signed = -1;
u16:unsigned = 100;

bool:comparison = (signed < unsigned);  // FALSE! (-1 → 65,535)
```

### 7. Array Size Calculations

```aria
u16:width = 1920;
u16:height = 1080;
u16:total = width * height;  // Overflows! (wraps to 41,216)

// CORRECT: Widen before multiply
u32:total_correct = width.to_u32() * height.to_u32();
```

## Summary

**u16** is the **medium unsigned integer** for 16-bit values:

**Use u16 when**:
- Network ports (TCP/UDP 0-65535)
- Image dimensions ≤65K pixels
- Unicode BMP code points
- Medium array sizes (256-65K)
- Checksums (CRC-16)
- Device/sensor IDs

**Avoid u16 when**:
- Values can be negative (use `i16`)
- Small values ≤255 (use `u8` to save memory)
- Large values >65,535 (use `u32`)
- Subtraction might underflow

**Key Principles**:
1. **Never negative** (always >= 0)
2. **Underflow wraps to 65,535** (0 - 1 = 65,535!)
3. **Check before subtracting**
4. **Standard for network ports**
5. **Widen before large multiplication**

**Strengths**:
- ✓ Perfect for network ports (industry standard)
- ✓ Image dimensions (reasonable sizes)
- ✓ Unicode BMP (most common characters)
- ✓ Medium arrays (up to 65K elements)
- ✓ Good SIMD width (8/16/32 at once)
- ✓ C compatible (`uint16_t`)

**Limitations**:
- ✗ No negative values
- ✗ Underflow wraps silently (0 - 1 = 65,535)
- ✗ Limited range (0-65,535 only)
- ✗ Mixing with signed is error-prone

The `u16` type is the **network port standard** and ideal for medium-sized non-negative values in Aria!
