# u32: 32-Bit Unsigned Integer Type

## Overview

**Purpose**: The `u32` type represents 32-bit unsigned integers for non-negative whole numbers. It's optimized for counts, sizes, indices, and values that are guaranteed never to be negative.

**Characteristics**:
- Size: 32 bits (4 bytes)
- Range: 0 to 4,294,967,295 (0 to ~4.3 billion)
- Always non-negative (unsigned)
- Wrapping overflow and underflow (modular arithmetic)
- Zero-overhead (native machine instructions)
- Binary representation: All 32 bits for magnitude (no sign bit)

**When to Use u32**:
- Array lengths and indices (0 to 4.3B elements)
- File sizes up to 4GB
- Counts and statistics (users, events, items)
- Memory addresses (32-bit systems)
- IPv4 addresses (32-bit representation)
- Hash values (CRC-32, FNV, etc.)
- Unix timestamps (until year 2106)
- RGB colors (0xAARRGGBB format)
- Always-positive quantities
- C interop (`uint32_t`, `unsigned int`)

**When NOT to Use u32**:
- Values that can be negative (use `i32` instead)
- Subtraction that might go below zero (danger: wraps to MAX!)
- Mixed arithmetic with signed types (implicit conversion issues)
- File sizes >4GB (use `u64`)
- Security-critical calculations (wrapping can create vulnerabilities)
- When overflow detection needed (use `tbb32` for bounds-checked)
- Loop counters that decrement to zero (easy to underflow)

## u32 vs i32: Signed vs Unsigned

The choice between `u32` (unsigned) and `i32` (signed) determines whether negative values are possible.

### Key Differences

| Aspect | u32 (Unsigned) | i32 (Signed) | Notes |
|--------|----------------|--------------|-------|
| **Range** | 0 to 4,294,967,295 | -2,147,483,648 to 2,147,483,647 | u32 has 2× positive range |
| **Negative Values** | ❌ Never | ✅ Yes | u32 wraps on underflow |
| **Overflow** | Wraps to 0 | Wraps to MIN | Both wrap (defined behavior) |
| **Underflow** | Wraps to MAX | Wraps to MAX | **Critical danger for u32!** |
| **Sign Bit** | No (all 32 bits for value) | Yes (bit 31) | u32 uses extra bit |
| **Comparisons** | Always >= 0 | Can be < 0 | `if (u32_val < 0)` always false |
| **Default** | No | Yes | Literals default to i32 |

### The Underflow Trap

**CRITICAL**: u32 subtraction can wrap unexpectedly:

```aria
// DANGER: Underflow wrapping
u32:count = 5;
count -= 10;  // Wraps to 4,294,967,291! (not -5)

// This is LEGAL but WRONG semantically
u32:balance = 100;
balance -= 200;  // Wraps to 4,294,967,196 (not -100)

// CORRECT: Use i32 for values that can go negative
i32:signed_balance = 100;
signed_balance -= 200;  // -100 (correct)
```

### When to Choose u32 Over i32

**Use u32 when**:
```aria
// Sizes and counts (never negative)
u32:array_length = data.length();
u32:user_count = database.count_users();
u32:pixel_count = width * height;

// Memory/file sizes up to 4GB
u32:buffer_size = allocate_buffer();
u32:file_size_bytes = file.size();

// Array indices (0-based)
loop (array.length():i) {
    u32:index = i;  // Always >= 0
    process(array[index]);
}

// Hash values (always positive)
u32:hash = fnv1a_hash(data);
u32:checksum = crc32(buffer);

// IPv4 addresses
u32:localhost = 0x7F000001;  // 127.0.0.1
```

**Use i32 when**:
```aria
// Values that might be negative
i32:temperature = -15;  // Celsius (can be negative)
i32:delta = end - start;  // Difference (might be negative)
i32:offset = -100;  // Relative position

// Subtraction results
i32:balance = deposits - withdrawals;  // Can go negative

// Signed comparisons
i32:value = get_value();
if (value < 0) {  // Meaningful check
    error("Negative value not allowed");
}

// General-purpose integers (default)
i32:count = 42;  // Use i32 unless u32 specifically needed
```

### Decision Matrix

```
Choose u32 when:
✓ Value CANNOT be negative (semantic guarantee)
✓ Need full 4.3 billion positive range (vs i32's 2.1B)
✓ Array sizes/indices (lengths always >= 0)
✓ Memory/file sizes up to 4GB
✓ C interop with unsigned types
✓ Bit patterns (hashes, colors, flags)

Choose i32 when:
✓ Value MIGHT be negative
✓ Subtraction involved (risk of underflow)
✓ General-purpose integer (default choice)
✓ Mixed arithmetic with signed types
✓ Range fits in ±2.1 billion

DANGER ZONE (use with care):
⚠ u32 loop counters that decrement
⚠ u32 subtraction without bounds checking
⚠ Mixing u32 and i32 in comparisons
⚠ u32 for balance/delta calculations
```

## u32 vs u64: Size Selection

Choosing between `u32` (4 bytes) and `u64` (8 bytes) depends on maximum value and performance considerations.

### Key Differences

| Aspect | u32 (32-bit) | u64 (64-bit) | Notes |
|--------|--------------|--------------|-------|
| **Range** | 0 to 4.3 billion | 0 to 18.4 quintillion | u64 vastly larger |
| **Size** | 4 bytes | 8 bytes | u32 uses half memory |
| **Max File Size** | 4GB | 18 exabytes | u64 for modern filesystems |
| **Array Limit** | 4.3B elements | 18.4 quintillion | Practical limit is RAM |
| **Performance** | Same on 64-bit CPUs | Same on 64-bit CPUs | No speed difference |
| **SIMD Width** | 4/8/16 values | 2/4/8 values | u32 better vectorization |
| **Memory** | 1× | 2× | Arrays take 2× space with u64 |

### When to Choose u32 Over u64

**File sizes up to 4GB**:
```aria
// 32-bit file size (old filesystems, legacy formats)
u32:file_size = 3_221_225_472;  // 3GB (fits in u32)

// Reading 32-bit WAV header
struct WAVHeader {
    u32:chunk_size;    // File size - 8 bytes
    u32:subchunk1_size;
    u32:sample_rate;
    u32:byte_rate;
}
```

**Array indices for medium-sized arrays**:
```aria
// Array with millions of elements (not billions)
u32:array_length = 10_000_000;  // 10 million
loop (array_length:i) {
    u32:index = i;
    process(array[index]);
}

// Hash table with 4.3B max buckets
u32:bucket_count = 1 << 20;  // 1 million buckets
u32:hash_index = hash_value % bucket_count;
```

**Memory-constrained environments**:
```aria
// Embedded system: Save memory with u32
struct SensorReading {
    u32:timestamp;     // 4 bytes (vs 8 for u64)
    u32:sample_count;  // 4 bytes
    u32:checksum;      // 4 bytes
}

// Array of 1 million readings
// u32: 12MB, u64: 24MB (2× memory)
SensorReading[1_000_000]:readings;
```

### When to Choose u64 Over u32

**Large file sizes (>4GB)**:
```aria
// Modern file sizes (videos, databases, backups)
u64:file_size = 549_755_813_888;  // 512GB
u64:total_bytes_transferred = 1_099_511_627_776;  // 1TB

// Filesystem operations
u64:disk_space = get_available_space();  // Might be >4GB
```

**High-resolution timestamps**:
```aria
// Nanosecond timestamps (u32 would overflow in seconds)
u64:timestamp_ns = get_timestamp_nanoseconds();

// Milliseconds since epoch (year 2554 vs 2106 for u32)
u64:timestamp_ms = get_timestamp_milliseconds();
```

**Large-scale counting**:
```aria
// Counting beyond 4.3 billion
u64:total_requests = 10_000_000_000;  // 10 billion
u64:bytes_processed = 5_000_000_000_000;  // 5 trillion
```

### Decision Matrix

```
Choose u32 when:
✓ Values guaranteed < 4.3 billion
✓ File sizes < 4GB
✓ Array lengths < 4.3B elements (practical limit ~1B)
✓ Memory matters (embedded, mobile)
✓ Legacy 32-bit compatibility
✓ IPv4 addresses (exactly 32 bits)

Choose u64 when:
✓ Files > 4GB (modern filesystems)
✓ Memory addresses (64-bit pointers)
✓ High-resolution timestamps
✓ Large counts (>4.3 billion)
✓ Future-proofing (avoid u32 limits)
✓ Byte counts for large transfers

Default recommendation:
- Arrays/indices: u32 (unless >4B elements)
- File sizes: u64 (modern default)
- Timestamps: u64 (milliseconds or better)
- Counters: u32 unless scale demands u64
```

## Range and Representation

### Numeric Range

**Minimum**: 0  
**Maximum**: 4,294,967,295 (2³² - 1)  
**Total values**: 4,294,967,296 (2³²)

**In different bases**:
```aria
u32:max_decimal = 4294967295;
u32:max_hex = 0xFFFFFFFF;
u32:max_binary = 0b11111111111111111111111111111111;

// Common boundaries
u32:zero = 0;
u32:one_billion = 1_000_000_000;
u32:two_billion = 2_000_000_000;
u32:max = 4_294_967_295;
```

### Binary Representation

All 32 bits represent magnitude (no sign bit):

```
Bit positions:  31 30 29 ... 2  1  0
Value:          0  0  0  ... 0  0  0  = 0
Value:          1  1  1  ... 1  1  1  = 4,294,967,295

Example: 42 in u32
Binary: 00000000 00000000 00000000 00101010
Hex:    0x0000002A
```

**Comparison with i32** (same 32 bits, different interpretation):
```aria
// Same bit pattern, different meaning
u32:unsigned_max = 0xFFFFFFFF;  // 4,294,967,295
i32:signed = -1;                 // Same bits: 0xFFFFFFFF

// Conversion (bit reinterpretation)
i32:negative = -1;
u32:as_unsigned = negative.as_u32();  // 4,294,967,295 (bit pattern unchanged)
```

## Overflow and Underflow

### Overflow (Addition Beyond MAX)

u32 wraps modulo 2³²:

```aria
// Overflow wrapping
u32:max = 4_294_967_295;
u32:overflow = max + 1;  // Wraps to 0

u32:near_max = 4_294_967_290;
near_max += 10;  // Wraps to 4 (4,294,967,290 + 10 - 2³²)

// Multiplication overflow
u32:large = 1_000_000;
u32:overflow_result = large * 10_000;  // Wraps (10B > 4.3B)

// Checking for overflow before operation
u32:safe_add(u32:a, u32:b) -> Result<u32> {
    if (a > u32.max - b) {
        return Error("Overflow would occur");
    }
    return Ok(a + b);
}
```

### Underflow (Subtraction Below Zero)

**CRITICAL DANGER**: u32 underflow wraps to MAX:

```aria
// Classic underflow bug
u32:count = 5;
count -= 10;  // Wraps to 4,294,967,291 (!!)

// Loop underflow trap
u32:index = 0;
index -= 1;  // Wraps to 4,294,967,295 (not -1!)

// BAD: Decrementing loop (underflows at 0)
u32:i = 10;
while (i >= 0) {  // ALWAYS TRUE! (u32 always >= 0)
    process(i);
    i -= 1;  // When i=0, becomes 4,294,967,295!
}
// Infinite loop!

// CORRECT: Use i32 for decrementing loops
i32:i = 10;
while (i >= 0) {
    process(i.to_u32());
    i -= 1;  // Correctly goes negative
}
```

### Detecting Under/Overflow

```aria
// Safe subtraction
u32:safe_sub(u32:a, u32:b) -> Result<u32> {
    if (b > a) {
        return Error("Underflow would occur");
    }
    return Ok(a - b);
}

// Check before arithmetic
u32:balance = 100;
u32:withdrawal = 200;

if (withdrawal > balance) {
    error("Insufficient funds");
} else {
    balance -= withdrawal;
}

// Overflow-checking addition
u32:a = 4_000_000_000;
u32:b = 500_000_000;

if (a > u32.max - b) {
    error("Addition would overflow");
} else {
    u32:sum = a + b;
}
```

## Operations

### Arithmetic

```aria
u32:a = 100;
u32:b = 50;

u32:sum = a + b;          // 150
u32:difference = a - b;   // 50 (safe: 100 > 50)
u32:product = a * b;      // 5000
u32:quotient = a / b;     // 2 (integer division)
u32:remainder = a % b;    // 0

// Compound assignment
a += b;   // a = 150
a -= b;   // a = 100
a *= 2;   // a = 200
a /= 10;  // a = 20
a %= 7;   // a = 6
```

### Division by Zero

```aria
u32:numerator = 100;
u32:denominator = 0;

// Runtime error (panic)
u32:result = numerator / denominator;  // ERROR!

// Safe division
u32:safe_divide(u32:a, u32:b) -> Result<u32> {
    if (b == 0) {
        return Error("Division by zero");
    }
    return Ok(a / b);
}

match (safe_divide(numerator, denominator)) {
    Ok(result) => { use(result); }
    Error(msg) => { error(msg); }
}
```

### Bitwise Operations

```aria
u32:a = 0b11110000;
u32:b = 0b10101010;

u32:and = a & b;      // 0b10100000 (bitwise AND)
u32:or = a | b;       // 0b11111010 (bitwise OR)
u32:xor = a ^ b;      // 0b01011010 (bitwise XOR)
u32:not = ~a;         // 0b11111111111111111111111100001111 (bitwise NOT)

// Bit shifts
u32:value = 0b1010;
u32:left = value << 2;   // 0b101000 (multiply by 4)
u32:right = value >> 1;  // 0b101 (divide by 2)

// Extracting bits (IPv4 octets)
u32:ip = 0xC0A80101;  // 192.168.1.1
u32:octet1 = (ip >> 24) & 0xFF;  // 192
u32:octet2 = (ip >> 16) & 0xFF;  // 168
u32:octet3 = (ip >> 8) & 0xFF;   // 1
u32:octet4 = ip & 0xFF;           // 1
```

### Comparisons

```aria
u32:a = 100;
u32:b = 50;

bool:equal = (a == b);        // false
bool:not_equal = (a != b);    // true
bool:greater = (a > b);       // true
bool:greater_equal = (a >= b); // true
bool:less = (a < b);          // false
bool:less_equal = (a <= b);    // false

// Always false (u32 never negative)
bool:is_negative = (a < 0);  // Always false!

// Checking for zero
bool:is_zero = (a == 0);
bool:is_nonzero = (a != 0);
```

### Min/Max

```aria
u32:a = 100;
u32:b = 50;

u32:maximum = max(a, b);  // 100
u32:minimum = min(a, b);  // 50

// Constants
u32:min_value = u32.min;  // 0
u32:max_value = u32.max;  // 4,294,967,295
```

## Type Conversions

### From Smaller Unsigned Types

Safe (no data loss):

```aria
// u8 → u32 (always safe)
u8:byte = 255;
u32:word = byte.to_u32();  // 255

// u16 → u32 (always safe)
u16:short = 65535;
u32:long = short.to_u32();  // 65'535
```

### To Smaller Unsigned Types

Truncates (data loss if too large):

```aria
// u32 → u8 (truncates to lowest 8 bits)
u32:value = 300;
u8:truncated = value.to_u8();  // 44 (300 mod 256)

// u32 → u16 (truncates to lowest 16 bits)
u32:big = 70000;
u16:small = big.to_u16();  // 4464 (70000 mod 65536)

// Safe conversion with bounds check
u32:safe_to_u8(u32:value) -> Result<u8> {
    if (value > 255) {
        return Error("Value too large for u8");
    }
    return Ok(value.to_u8());
}
```

### From Signed Types

Bit reinterpretation (negative becomes large positive):

```aria
// i32 → u32 (bit pattern unchanged)
i32:negative = -1;
u32:as_unsigned = negative.to_u32();  // 4,294,967,295

i32:small_negative = -100;
u32:wrapped = small_negative.to_u32();  // 4,294,967,196

// Safe conversion (reject negatives)
i32:safe_to_u32(i32:value) -> Result<u32> {
    if (value < 0) {
        return Error("Cannot convert negative to unsigned");
    }
    return Ok(value.to_u32());
}
```

### To Signed Types

Wraps if > i32.max:

```aria
// u32 → i32 (values >2.1B wrap to negative)
u32:small = 100;
i32:safe = small.to_i32();  // 100

u32:large = 3_000_000_000;  // 3 billion
i32:wrapped = large.to_i32();  // Negative! (wraps)

// Safe conversion
u32:safe_to_i32(u32:value) -> Result<i32> {
    if (value > i32.max.to_u32()) {
        return Error("Value too large for i32");
    }
    return Ok(value.to_i32());
}
```

### To Larger Unsigned Types

Always safe (zero-extension):

```aria
// u32 → u64 (always safe)
u32:value = 4_000_000_000;
u64:extended = value.to_u64();  // 4,000,000,000 (exact)
```

// Natural fit for array indices
u32[]:data = [10, 20, 30, 40, 50];

loop (length:i) {

u32:safe_get(u32[]:array, u32:index) -> Result<u32> {

}

// Reading file metadata
struct FileInfo {

FileInfo:read_file_info(string:path) -> Result<FileInfo> {
    // ... filesystem API ...

// Check if file fits in buffer

### Hash Functions

```aria
// FNV-1a hash (32-bit)
u32:fnv1a_hash(u8[]:data) -> u32 {
    u32:hash = 2166136261;  // FNV offset basis
    u32:prime = 16777619;   // FNV prime
    
    loop (data.length():i) {
        hash ^= data[i].to_u32();
        hash *= prime;  // Wraps on overflow (intentional)
    }
    
    return hash;
}

// CRC-32 checksum
u32:crc32(u8[]:data) -> u32 {
    u32:crc = 0xFFFFFFFF;
    
    loop (data.length():i) {
        u32:byte = data[i].to_u32();
        crc ^= byte;
        
        loop (8:bit) {
            u32:mask = (crc & 1) ? 0xFFFFFFFF : 0;
            crc = (crc >> 1) ^ (0xEDB88320 & mask);
        }
    }
    
    return ~crc;
}
```

### IPv4 Address Manipulation

```aria
// IPv4 as u32
struct IPv4 {
    u32:address;
}

IPv4:from_octets(u8:a, u8:b, u8:c, u8:d) -> IPv4 {
    u32:addr = (a.to_u32() << 24)
             | (b.to_u32() << 16)
             | (c.to_u32() << 8)
             | d.to_u32();
    return IPv4 { address: addr };
}

string:to_string(IPv4:ip) -> string {
    u8:a = ((ip.address >> 24) & 0xFF).to_u8();
    u8:b = ((ip.address >> 16) & 0xFF).to_u8();
    u8:c = ((ip.address >> 8) & 0xFF).to_u8();
    u8:d = (ip.address & 0xFF).to_u8();
    
    return "${a}.${b}.${c}.${d}";
}

// Example usage
IPv4:localhost = from_octets(127, 0, 0, 1);
IPv4:private = from_octets(192, 168, 1, 100);

print(to_string(localhost));  // "127.0.0.1"
print(to_string(private));    // "192.168.1.100"
```

## Anti-Patterns

### ❌ Underflow in Loops

```aria
// WRONG: Decrementing u32 loop (underflows!)
u32:i = 10;
while (i >= 0) {  // ALWAYS TRUE!
    process(i);
    i -= 1;  // At i=0, wraps to 4,294,967,295
}
// Infinite loop!

// CORRECT: Use i32 for decrementing
i32:i = 10;
while (i >= 0) {
    process(i.to_u32());
    i -= 1;  // Goes to -1, exits loop
}

// ALTERNATIVE: Count up instead
u32:count = 10;
loop (count:i) {
    u32:reverse_index = count - i - 1;
    process(reverse_index);
}
```

### ❌ Using u32 for Values That Can Be Negative

```aria
// WRONG: Temperature in u32
u32:temperature = -5;  // Wraps to 4,294,967,291!

// WRONG: Balance that can be negative
u32:balance = 100;
balance -= 200;  // Wraps to 4,294,967,196 (not -100)

// CORRECT: Use i32 for signed values
i32:temperature = -5;  // Correct
i32:balance = 100;
balance -= 200;  // -100 (correct)
```

### ❌ Mixing Signed and Unsigned Without Checks

```aria
// WRONG: Comparing signed and unsigned
i32:signed_value = -10;
u32:unsigned_value = 5;

if (signed_value < unsigned_value) {  // DANGER!
    // Implicit conversion: -10 becomes 4,294,967,286
    // Comparison: 4,294,967,286 < 5 is FALSE!
}

// CORRECT: Explicit conversion with validation
if (signed_value < 0 || signed_value.to_u32() < unsigned_value) {
    // Handles negative correctly
}
```

### ❌ Ignoring Overflow in Accumulation

```aria
// WRONG: No overflow checking
u32:total = 0;
loop (data.length():i) {
    total += data[i];  // Might overflow silently
}

// CORRECT: Check for overflow
u32:total = 0;
loop (data.length():i) {
    if (total > u32.max - data[i]) {
        error("Accumulation overflow");
        break;
    }
    total += data[i];
}

// ALTERNATIVE: Use u64 for accumulation
u64:total = 0;
loop (data.length():i) {
    total += data[i].to_u64();  // No overflow up to 18 quintillion
}
```

### ❌ Using u32 for Differences

```aria
// WRONG: Unsigned difference (can underflow)
u32:start_time = 1000;
u32:end_time = 500;
u32:duration = end_time - start_time;  // Wraps to huge number!

// CORRECT: Use signed for differences
i64:duration = end_time.to_i64() - start_time.to_i64();
if (duration < 0) {
    error("Invalid time range");
}
```

## Best Practices

### ✓ Use u32 for Sizes and Counts

```aria
// Array lengths
u32:length = array.length();

// User counts
u32:active_users = database.count_users();

// Byte counts (up to 4GB)
u32:bytes_read = file.read(buffer);
```

### ✓ Check Bounds Before Subtraction

```aria
// Always validate before subtracting
u32:balance = 100;
u32:withdrawal = 50;

if (withdrawal > balance) {
    error("Insufficient balance");
} else {
    balance -= withdrawal;  // Safe
}
```

### ✓ Use Explicit Conversions

```aria
// Make conversions explicit and checked
i32:signed_value = -10;

if (signed_value < 0) {
    error("Cannot convert negative to u32");
} else {
    u32:unsigned = signed_value.to_u32();
}
```

### ✓ Prefer u64 for Accumulation

```aria
// Counts that might exceed 4.3B
u64:total_bytes = 0;
loop (files.length():i) {
    total_bytes += files[i].size.to_u64();
}
```

### ✓ Document Underflow Invariants

```aria
// Document assumptions clearly
// PRECONDITION: new_value >= current_value
void:update_counter(u32&:current, u32:new_value) {
    assert(new_value >= current, "Counter must not decrease");
    current = new_value;
}
```

### ✓ Use Constants for Clarity

```aria
// Named constants better than magic numbers
u32:KILOBYTE = 1024;
u32:MEGABYTE = 1024 * KILOBYTE;
u32:MAX_BUFFER_SIZE = 4 * MEGABYTE;

u32:buffer_size = min(file_size, MAX_BUFFER_SIZE);
```

## Performance Characteristics

### CPU Operations

**Native 32-bit instructions** on all modern CPUs:

| Operation | Cycles | Notes |
|-----------|--------|-------|
| Addition/Subtraction | 1 | Fully pipelined |
| Multiplication | 3-5 | Fast on modern CPUs |
| Division | 10-30 | Slow, avoid in hot loops |
| Bitwise ops | 1 | AND, OR, XOR, NOT |
| Bit shifts | 1 | Left, right shifts |
| Comparisons | 1 | Equal, less, greater |

### SIMD Performance

**Vectorization** (process multiple u32 values in parallel):

- **SSE (128-bit)**: Process 4× u32 values per instruction
- **AVX2 (256-bit)**: Process 8× u32 values per instruction
- **AVX-512 (512-bit)**: Process 16× u32 values per instruction

```aria
// SIMD-friendly: Compilers can auto-vectorize
void:add_arrays(u32[]:a, u32[]:b, u32[]:result) {
    loop (a.length():i) {
        result[i] = a[i] + b[i];  // Vectorizes to 4/8/16 at a time
    }
}
```

### Memory Efficiency

**Size**: 4 bytes per value

```aria
// Array memory usage
u32[1_000_000]:data;  // 4MB

// vs u64
u64[1_000_000]:data;  // 8MB (2× memory)

// vs i32 (same size)
i32[1_000_000]:data;  // 4MB (same as u32)
```

**Cache efficiency**: u32 arrays fit 2× more values per cache line than u64.

## Common Gotchas

### 1. Underflow Wrapping (Most Common!)

```aria
u32:count = 5;
count -= 10;  // Wraps to 4,294,967,291 (not -5!)

// Always check before subtraction
if (decrement > count) {
    error("Underflow");
} else {
    count -= decrement;
}
```

### 2. Always >= 0 Checks

```aria
u32:value = get_value();
if (value < 0) {  // ALWAYS FALSE!
    // This code never executes
}

// Use i32 if you need to check for negative
```

### 3. Overflow in Multiplication

```aria
u32:width = 100_000;
u32:height = 100_000;
u32:pixels = width * height;  // Overflows! (10B > 4.3B)

// Use u64 for intermediate calculation
u64:pixels = width.to_u64() * height.to_u64();  // Correct
```

### 4. Mixing with Signed Types

```aria
i32:signed = -1;
u32:unsigned = 10;

// Implicit conversion of signed to unsigned
bool:comparison = (signed < unsigned);  // FALSE! (-1 → 4.3B)

// Always explicit
bool:safe = (signed < 0) || (signed.to_u32() < unsigned);
```

### 5. Loop Decrement Trap

```aria
// Infinite loop (underflows at i=0)
u32:i = 10;
while (i >= 0) {
    process(i);
    i -= 1;  // Wraps to 4,294,967,295 at i=0
}

// Use i32 or count up
```

### 6. Division Truncates

```aria
u32:a = 7;
u32:b = 3;
u32:result = a / b;  // 2 (not 2.333...)

// Use f32/f64 for fractional results
f32:quotient = a.to_f32() / b.to_f32();  // 2.333...
```

### 7. Modulo with Zero

```aria
u32:value = 100;
u32:mod = value % 0;  // ERROR! (runtime panic)

// Always check divisor
if (divisor != 0) {
    u32:remainder = value % divisor;
}
```

## Summary

**u32** is the **unsigned 32-bit integer** for non-negative values:

**Use u32 when**:
- Array sizes/indices (0 to 4.3B)
- File sizes up to 4GB
- Counts, statistics
- Hash values, checksums
- IPv4 addresses
- Values guaranteed >= 0

**Avoid u32 when**:
- Values can be negative (use `i32`)
- Subtraction might underflow (use `i32`)
- Files >4GB (use `u64`)
- Mixed signed/unsigned arithmetic

**Key Principles**:
1. **Never negative** (always >= 0)
2. **Underflow wraps to MAX** (critical danger!)
3. **Check before subtracting** (prevent underflow)
4. **4.3 billion max** (use u64 for larger)
5. **Decrementing loops risky** (use i32 or count up)

**Strengths**:
- ✓ 2× positive range vs i32
- ✓ Zero-overhead performance
- ✓ Perfect for sizes/counts
- ✓ Natural for bit patterns
- ✓ SIMD vectorization

**Limitations**:
- ✗ No negative values
- ✗ Underflow wraps silently
- ✗ 4GB file size limit
- ✗ Mixing with signed is error-prone

The `u32` type is the **standard unsigned integer** for always-positive values up to 4.3 billion. Just watch out for underflow!
