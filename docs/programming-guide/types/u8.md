# u8: 8-Bit Unsigned Integer Type

## Overview

**Purpose**: The `u8` type represents 8-bit unsigned integers - the fundamental **byte** type. It's the universal unit for binary data, raw memory, networking, and file I/O.

**Characteristics**:
- Size: 8 bits (1 byte)
- Range: 0 to 255 (0 to 2⁸ - 1)
- Always non-negative (unsigned)
- Wrapping overflow and underflow (modular arithmetic)
- Zero-overhead (single machine instruction)
- Binary representation: All 8 bits for magnitude (no sign bit)

**When to Use u8**:
- Raw binary data (byte arrays, buffers)
- File I/O and networking (packets, protocols)
- Image data (RGB channels 0-255, grayscale pixels)
- ASCII characters (byte values 0-127)
- Small counts and percentages (0-100 scaled to 0-255)
- Bit manipulation (flags, masks, bitfields)
- C interop (`uint8_t`, `unsigned char`)
- Memory-constrained environments (embedded systems)

**When NOT to Use u8**:
- Values that can be negative (use `i8` instead)
- Unicode characters (use `char` for proper UTF-8)
- Values >255 (use `u16`, `u32`, or `u64`)
- Subtraction that might go below zero (underflow wraps to 255!)
- Mixed arithmetic with signed types (conversion issues)
- Loop counters that decrement to zero (underflow trap)

## u8 vs char: Binary vs Unicode

The choice between `u8` (byte) and `char` (Unicode) depends on whether you're working with **binary data** or **text**.

### Key Differences

| Aspect | u8 (Byte) | char (Unicode) | Notes |
|--------|-----------|----------------|-------|
| **Purpose** | Binary data | Unicode text | Semantic distinction |
| **Range** | 0 to 255 | U+0000 to U+10FFFF | char is 21-bit code point |
| **Size** | Always 1 byte | 1-4 bytes (UTF-8) | char is variable-width |
| **Encoding** | None (raw bits) | UTF-8 encoded | char handles encoding |
| **Operations** | Bitwise, numeric | Character classification | Different semantics |
| **ASCII 'A'** | 65 (number) | 'A' (character) | Values vs characters |

### When to Choose u8 Over char

**Raw binary data**:
```aria
// Network packet (binary protocol)
u8[1500]:packet;  // MTU-sized buffer
u8:header_byte = packet[0];  // Raw byte value

// File signature check
u8[4]:magic = read_bytes(file, 4);
if (magic[0] == 0x89 && magic[1] == 0x50 &&
    magic[2] == 0x4E && magic[3] == 0x47) {
    // PNG file signature
}
```

**Image pixel data**:
```aria
// RGB image (raw bytes, not text!)
struct RGBPixel {
    u8:red;    // 0-255
    u8:green;  // 0-255
    u8:blue;   // 0-255
}

u8:grayscale = 128;  // 50% gray (0=black, 255=white)
```

**Bit manipulation**:
```aria
// Status flags (bit patterns, not characters)
u8:flags = 0b1010_0001;
u8:mask = 0x0F;  // Lower nibble mask

flags |= 0b0000_0010;  // Set bit 1
flags &= ~0b0000_0100;  // Clear bit 2
```

### When to Choose char Over u8

**Text data**:
```aria
// Unicode characters (text, not bytes)
char:letter = 'A';  // Unicode U+0041
char:emoji = '😀';  // Unicode U+1F600 (4 bytes in UTF-8!)

// DON'T use u8 for text!
u8:bad = 65;  // This is the NUMBER 65, not the CHARACTER 'A'
```

**String processing**:
```aria
// char for character operations
string:text = "Hello, 世界";
loop (text.chars():c) {
    if (c.is_alpha()) {
        print("${c} is alphabetic\n");
    }
}

// u8[] for binary data
u8[]:bytes = text.as_bytes();  // Get underlying UTF-8 bytes
```

### Decision Matrix

```
Choose u8 when:
✓ Binary data (files, networks, memory)
✓ Image pixels (RGB channels, grayscale)
✓ Raw byte values (no encoding)
✓ Bit manipulation (flags, masks)
✓ Numeric byte operations
✓ C interop with uint8_t

Choose char when:
✓ Unicode text (letters, symbols, emoji)
✓ Character classification (is_alpha, is_digit)
✓ Case conversion (to_upper, to_lower)
✓ Human-readable text
✓ UTF-8 encoded content
✓ String building blocks

Rule: Binary data → u8, Text data → char
```

## u8 vs i8: Signed vs Unsigned Bytes

The choice between `u8` (unsigned) and `i8` (signed) determines whether negative values are possible.

### Key Differences

| Aspect | u8 (Unsigned) | i8 (Signed) | Notes |
|--------|---------------|-------------|-------|
| **Range** | 0 to 255 | -128 to +127 | u8 has 2× positive range |
| **Negative Values** | ❌ Never | ✅ Yes | u8 wraps on underflow |
| **Overflow** | Wraps to 0 | Wraps to MIN | Both defined behavior |
| **Underflow** | Wraps to 255 | Wraps to MAX | **Critical danger for u8!** |
| **Sign Bit** | No (all 8 bits for value) | Yes (bit 7) | u8 uses extra bit |
| **ASCII Range** | 0-127 standard, 128-255 extended | -128 to +127 | u8 covers full ASCII |

### The Underflow Trap (Critical!)

**DANGER**: u8 underflow creates large positive values:

```aria
// DANGER: Underflow wrapping
u8:count = 5;
count -= 10;  // Wraps to 251! (not -5)

// Even more dangerous in comparisons
u8:balance = 50;
balance -= 100;  // Wraps to 206 (looks like plenty left!)

if (balance > 10) {
    // Executes! balance is 206, not -50
    error("Unexpected large balance");
}

// CORRECT: Use i8 for values that can be negative
i8:balance_signed = 50;
balance_signed -= 100;  // -50 (correct)

if (balance_signed < 0) {
    error("Insufficient balance");
}
```

### When to Choose u8 Over i8

**Binary data** (never negative):
```aria
// File I/O (bytes can't be negative)
u8[1024]:buffer;
u64:bytes_read = file.read(buffer);

// Network protocols
u8:version = 4;  // IPv4
u8:header_length = 20;
```

**Image pixels** (0-255 range):
```aria
// RGB color
struct Color {
    u8:red;    // 0 (none) to 255 (full)
    u8:green;
    u8:blue;
}

Color:white = Color { red: 255, green: 255, blue: 255 };
```

**Small counts** (semantically non-negative):
```aria
// Age, quantity, percentage
u8:age = 25;
u8:quantity = 100;
u8:percentage = 75;  // 0-100% (or 0-255 scaled)
```

### When to Choose i8 Over u8

**Values that might be negative**:
```aria
// Temperature (can go below zero)
i8:temp_celsius = -5;  // -5°C

// Coordinate offsets (can be negative)
i8:x_offset = -10;  // Move left
i8:y_offset = 15;   // Move down

// Differences (subtraction results)
i8:delta = new_value - old_value;  // Might be negative
```

**Subtraction-heavy code**:
```aria
// Difference calculations
i8:start = 50;
i8:end = 30;
i8:difference = end - start;  // -20 (correct)

// vs u8
u8:start_u = 50;
u8:end_u = 30;
u8:difference_u = end_u - start_u;  // 236! (wrapped, wrong)
```

### Decision Matrix

```
Choose u8 when:
✓ Value CANNOT be negative (bytes, counts, pixels)
✓ Need full 0-255 positive range
✓ Binary data operations
✓ Image/color channels
✓ C interop with uint8_t

Choose i8 when:
✓ Value MIGHT be negative
✓ Temperature, offsets, deltas
✓ Subtraction involved
✓ Range fits in -128 to +127
✓ General-purpose small integer

Default: i8 for general use, u8 for bytes/binary
```

## u8 vs u16/u32: Size Selection

The choice between `u8`, `u16`, and `u32` depends on maximum value needed and memory considerations.

### Range Comparison

| Type | Bytes | Range | Use Cases |
|------|-------|-------|-----------|
| **u8** | 1 | 0 to 255 | Bytes, pixels, small counts |
| **u16** | 2 | 0 to 65,535 | Ports, large dimensions, IDs |
| **u32** | 4 | 0 to 4.3 billion | File sizes <4GB, large counts |
| **u64** | 8 | 0 to 18.4 quintillion | Modern files, addresses |

### When to Choose u8 Over Larger Types

**Values ≤255**:
```aria
// RGB channels (always 0-255)
u8:red = 200;
u8:green = 150;
u8:blue = 100;

// Age (reasonable assumption: <256 years)
u8:age = 25;

// ASCII codes (0-127 standard, 128-255 extended)
u8:ascii_A = 65;
```

**Memory efficiency**:
```aria
// Large array: Memory matters!
u8[1_000_000]:pixel_data;  // 1 MB

// vs u32
u32[1_000_000]:wasted;  // 4 MB (4× larger!)

// Storage: 1 million RGB pixels
struct RGB { u8:r; u8:g; u8:b; }
RGB[1_000_000]:image;  // 3 MB

// vs u32 channels
struct RGB32 { u32:r; u32:g; u32:b; }
RGB32[1_000_000]:wasteful;  // 12 MB!
```

**Cache performance**:
```aria
// Cache line (64 bytes) fits:
// - 64× u8 values
// - 32× u16 values
// - 16× u32 values
// - 8× u64 values

// More data per cache line = better performance
```

### When to Choose Larger Types

**u16** (values 256-65,535):
```aria
// Network ports
u16:http_port = 80;
u16:https_port = 443;

// Image dimensions
u16:width = 1920;   // 1920 pixels
u16:height = 1080;  // 1080 pixels
```

**u32** (values >65,535):
```aria
// Large counts
u32:population = 330_000_000;  // 330 million

// File sizes <4GB
u32:file_size = 2_147_483_648;  // 2GB
```

**u64** (values >4.3 billion):
```aria
// Modern file sizes
u64:video_size = 549_755_813_888;  // 512GB
```

### Decision Matrix

```
Choose u8 when:
✓ Values guaranteed ≤255
✓ Memory matters (large arrays)
✓ Cache performance critical
✓ Binary data (bytes)
✓ Image pixels (RGB channels)

Choose u16 when:
✓ Values 256-65,535
✓ Network ports
✓ Large dimensions
✓ Unicode BMP code points

Choose u32 when:
✓ Values >65,535
✓ Large counts
✓ File sizes <4GB
✓ Array indices (billions)

Choose u64 when:
✓ Files >4GB
✓ Memory addresses
✓ Timestamps (nanoseconds)

Smallest type that fits: Better memory, better cache
```

## Range and Representation

### Numeric Range

**Minimum**: 0  
**Maximum**: 255 (2⁸ - 1)  
**Total values**: 256 (2⁸)

**In different bases**:
```aria
u8:max_decimal = 255;
u8:max_hex = 0xFF;
u8:max_binary = 0b1111_1111;

// Common boundaries
u8:zero = 0;
u8:ascii_max = 127;  // Standard ASCII
u8:extended_max = 255;  // Extended ASCII
```

### Binary Representation

All 8 bits represent magnitude (no sign bit):

```
Bit positions:  7  6  5  4  3  2  1  0
Value:          0  0  0  0  0  0  0  0  = 0
Value:          1  1  1  1  1  1  1  1  = 255

Example: 65 ('A' in ASCII)
Binary: 0100_0001
Hex:    0x41
```

**Comparison with i8** (same 8 bits, different interpretation):
```aria
// Same bit pattern, different meaning
u8:unsigned_max = 0xFF;  // 255
i8:signed = -1;          // Same bits: 0xFF

// Conversion (bit reinterpretation)
i8:negative = -128;
u8:as_unsigned = negative.to_u8();  // 128
```

## Overflow and Underflow

### Overflow (Addition Beyond 255)

u8 wraps modulo 256:

```aria
// Overflow wrapping
u8:max = 255;
u8:overflow = max + 1;  // Wraps to 0

u8:value = 250;
value += 10;  // Wraps to 4 (250 + 10 = 260 % 256 = 4)

// Multiplication overflow
u8:large = 20;
u8:result = large * 20;  // 400 % 256 = 144 (wrapped)

// Checking for overflow before operation
u8:safe_add(u8:a, u8:b) -> Result<u8> {
    if (a > 255 - b) {
        return Error("Overflow would occur");
    }
    return Ok(a + b);
}
```

### Underflow (Subtraction Below Zero)

**CRITICAL DANGER**: u8 underflow creates large values:

```aria
// Classic underflow bug
u8:count = 5;
count -= 10;  // Wraps to 251 (not -5!)

// Real-world scenario
u8:hp = 30;  // Health points
u8:damage = 50;
hp -= damage;  // Wraps to 236 (looks alive with lots of HP!)

if (hp > 0) {
    print("Still alive with ${hp} HP\n");  // Executes!
}

// CORRECT: Check before subtraction
if (damage >= hp) {
    hp = 0;  // Dead
    print("You died!\n");
} else {
    hp -= damage;  // Safe
    print("${hp} HP remaining\n");
}
```

### Loop Underflow Trap

```aria
// WRONG: Decrementing u8 loop (infinite!)
u8:i = 10;
while (i >= 0) {  // ALWAYS TRUE (u8 always >= 0)
    process(i);
    i -= 1;  // At i=0, wraps to 255
}

// CORRECT: Use i8 or count up
i8:i = 10;
while (i >= 0) {
    process(i.to_u8());
    i -= 1;  // Goes to -1, exits loop
}

// ALTERNATIVE: Count up
u8:count = 10;
loop (count:i) {
    u8:reverse = count - i - 1;
    process(reverse);
}
```

## Operations

### Arithmetic

```aria
u8:a = 100;
u8:b = 50;

u8:sum = a + b;          // 150
u8:difference = a - b;   // 50 (safe: a > b)
u8:product = a * 2;      // 200
u8:quotient = a / b;     // 2
u8:remainder = a % b;    // 0

// Compound assignment
a += b;   // 150
a -= b;   // 100
a *= 2;   // 200
a /= 10;  // 20
a %= 7;   // 6
```

### Bitwise Operations

**Perfect for bit manipulation**:

```aria
u8:a = 0b1010_1010;
u8:b = 0b1100_1100;

u8:and = a & b;      // 0b1000_1000 (AND)
u8:or = a | b;       // 0b1110_1110 (OR)
u8:xor = a ^ b;      // 0b0110_0110 (XOR)
u8:not = ~a;         // 0b0101_0101 (NOT/flip)

// Bit shifts
u8:value = 0b0000_0001;  // 1
u8:left = value << 1;    // 0b0000_0010 (2)
u8:left7 = value << 7;   // 0b1000_0000 (128, NOT negative!)

// Right shift (logical - zero-fill)
u8:high = 0b1100_0000;  // 192
u8:right = high >> 1;   // 0b0110_0000 (96, zero-filled!)
```

### Bit Manipulation Patterns

```aria
// Set bit
u8:flags = 0;
flags |= (1 << 3);  // Set bit 3: 0b0000_1000

// Clear bit
flags &= ~(1 << 3);  // Clear bit 3

// Toggle bit
flags ^= (1 << 3);  // Toggle bit 3

// Test bit
if ((flags & (1 << 3)) != 0) {
    print("Bit 3 is set\n");
}

// Extract nibbles (4-bit groups)
u8:value = 0xA5;
u8:high_nibble = (value >> 4) & 0x0F;  // 0xA (10)
u8:low_nibble = value & 0x0F;          // 0x5 (5)
```

### Comparisons

```aria
u8:a = 100;
u8:b = 50;

bool:equal = (a == b);        // false
bool:not_equal = (a != b);    // true
bool:greater = (a > b);       // true
bool:less = (a < b);          // false

// Always false (u8 never negative)
bool:is_negative = (a < 0);  // Always false!

// Checking for zero
bool:is_zero = (a == 0);
bool:is_nonzero = (a != 0);
```

## Type Conversions

### From Smaller Types (Always Safe)

No smaller unsigned type exists! u8 is the smallest.

### To Larger Types (Always Safe)

```aria
// u8 → u16 (always safe)
u8:byte = 255;
u16:extended = byte.to_u16();  // 255

// u8 → u32 (always safe)
u32:big = byte.to_u32();  // 255

// u8 → u64 (always safe)
u64:huge = byte.to_u64();  // 255
```

### From Larger Types (Truncates)

```aria
// u16 → u8 (truncates to lowest 8 bits)
u16:large = 300;
u8:truncated = large.to_u8();  // 44 (300 % 256)

// u32 → u8 (truncates)
u32:value = 1000;
u8:byte = value.to_u8();  // 232 (1000 % 256)

// Safe conversion with bounds check
u32:safe_to_u8(u32:value) -> Result<u8> {
    if (value > 255) {
        return Error("Value too large for u8");
    }
    return Ok(value.to_u8());
}
```

### From Signed Types

Bit reinterpretation (negative becomes positive):

```aria
// i8 → u8 (bit pattern unchanged)
i8:negative = -1;
u8:as_unsigned = negative.to_u8();  // 255

i8:small_neg = -128;
u8:wrapped = small_neg.to_u8();  // 128

// Safe conversion (reject negatives)
i8:safe_to_u8(i8:value) -> Result<u8> {
    if (value < 0) {
        return Error("Cannot convert negative to unsigned");
    }
    return Ok(value.to_u8());
}
```

### To Signed Types

Wraps if >127:

```aria
// u8 → i8 (values >127 wrap to negative)
u8:small = 100;
i8:safe = small.to_i8();  // 100

u8:large = 200;
i8:wrapped = large.to_i8();  // -56 (wrapped)

// Safe conversion
u8:safe_to_i8(u8:value) -> Result<i8> {
    if (value > 127) {
        return Error("Value too large for i8");
    }
    return Ok(value.to_i8());
}
```

## Common Patterns

### Byte Buffers and File I/O

```aria
// Reading file data
u8[4096]:buffer;  // 4KB buffer
u64:bytes_read = file.read(buffer);

loop (bytes_read:i) {
    u8:byte = buffer[i];
    process_byte(byte);
}

// Writing binary data
u8[]:data = [0x4D, 0x5A, 0x90, 0x00];  // EXE signature
file.write(data);
```

### Image Processing

```aria
// RGB pixel manipulation
struct Pixel {
    u8:red;
    u8:green;
    u8:blue;
}

// Brighten pixel (saturating addition)
void:brighten(Pixel&:pixel, u8:amount) {
    u16:new_red = pixel.red.to_u16() + amount.to_u16();
    pixel.red = new_red > 255 ? 255 : new_red.to_u8();
    
    u16:new_green = pixel.green.to_u16() + amount.to_u16();
    pixel.green = new_green > 255 ? 255 : new_green.to_u8();
    
    u16:new_blue = pixel.blue.to_u16() + amount.to_u16();
    pixel.blue = new_blue > 255 ? 255 : new_blue.to_u8();
}

// Grayscale conversion
u8:to_grayscale(Pixel:pixel) -> u8 {
    // Weighted average (perceived brightness)
    u32:gray = (pixel.red.to_u32() * 299 +
                pixel.green.to_u32() * 587 +
                pixel.blue.to_u32() * 114) / 1000;
    return gray.to_u8();
}
```

### Network Protocols

```aria
// IPv4 packet processing
struct IPv4Header {
    u8:version_ihl;     // Version (4 bits) + IHL (4 bits)
    u8:dscp_ecn;        // DSCP (6 bits) + ECN (2 bits)
    u16:total_length;
    u16:identification;
    u16:flags_offset;
    u8:ttl;
    u8:protocol;
    u16:checksum;
    u8[4]:source_ip;
    u8[4]:dest_ip;
}

// Extract version from packed byte
u8:get_version(u8:version_ihl) -> u8 {
    return (version_ihl >> 4) & 0x0F;  // Upper 4 bits
}

// Extract IHL
u8:get_ihl(u8:version_ihl) -> u8 {
    return version_ihl & 0x0F;  // Lower 4 bits
}
```

### Checksums and Hashing

```aria
// Simple checksum (sum of all bytes)
u8:checksum_8bit(u8[]:data) -> u8 {
    u8:sum = 0;
    loop (data.length():i) {
        sum += data[i];  // Wrapping addition (intentional)
    }
    return sum;
}

// XOR checksum
u8:xor_checksum(u8[]:data) -> u8 {
    u8:result = 0;
    loop (data.length():i) {
        result ^= data[i];
    }
    return result;
}
```

### Saturating Arithmetic

```aria
// Saturating addition (clamp at 255, don't wrap)
u8:saturating_add(u8:a, u8:b) -> u8 {
    u16:sum = a.to_u16() + b.to_u16();
    return sum > 255 ? 255 : sum.to_u8();
}

// Saturating subtraction (clamp at 0, don't wrap)
u8:saturating_sub(u8:a, u8:b) -> u8 {
    if (b > a) {
        return 0;  // Prevent underflow wrap
    }
    return a - b;
}
```

## Anti-Patterns

### ❌ Underflow in Loops

```aria
// WRONG: Decrementing u8 loop (infinite!)
u8:i = 10;
while (i >= 0) {  // ALWAYS TRUE!
    process(i);
    i -= 1;  // At i=0, wraps to 255
}

// CORRECT: Use i8 or count up
i8:i = 10;
while (i >= 0) {
    process(i.to_u8());
    i -= 1;
}
```

### ❌ Using u8 for Text

```aria
// WRONG: u8 for text (loses Unicode support)
u8[]:bad_text = [72, 101, 108, 108, 111];  // "Hello" in ASCII
// Can't represent emoji, Chinese, etc.

// CORRECT: Use char/string for text
string:good_text = "Hello 世界 😀";
```

### ❌ Ignoring Underflow in Subtraction

```aria
// WRONG: No underflow checking
u8:hp = 30;
hp -= 50;  // Wraps to 236 (looks alive!)

// CORRECT: Check before subtracting
if (damage >= hp) {
    hp = 0;
} else {
    hp -= damage;
}
```

### ❌ Using u8 for Large Values

```aria
// WRONG: Value won't fit
u8:population = 10_000;  // Wraps to 16 (10000 % 256)

// CORRECT: Use larger type
u32:population = 10_000;
```

### ❌ Mixing Signed and Unsigned

```aria
// WRONG: Implicit conversion
i8:signed = -1;
u8:unsigned = 10;

if (signed < unsigned) {  // DANGER!
    // -1 converts to 255 unsigned
    // Comparison: 255 < 10 is FALSE!
}

// CORRECT: Explicit handling
if (signed < 0 || signed.to_u8() < unsigned) {
    // Properly handles negative
}
```

```aria
// Perfect for raw bytes
```

```aria
// Validate subtraction won't underflow
if (b > a) {
```

```aria
// Make all conversions explicit
i8:signed_value = get_signed_value();

if (signed_value < 0) {

    u8:unsigned = signed_value.to_u8();
```

```aria
// Prevent wrapping in pixel operations
u8:brighten_pixel(u8:original, u8:amount) {
```

```aria
// Document expected ranges
// EXPECTS: percentage is 0-100
u8:scale_percentage(u8:percentage) -> u8 {

}
```

## Performance Characteristics

### CPU Operations

**Native 8-bit instructions**:

| Operation | Cycles | Notes |
|-----------|--------|-------|
| Addition/Subtraction | 1 | Fully pipelined |
| Multiplication | 3-5 | Fast on modern CPUs |
| Division | 10-30 | Slower, avoid in hot loops |
| Bitwise ops | 1 | AND, OR, XOR, NOT |
| Bit shifts | 1 | Left, right shifts |
| Comparisons | 1 | Equal, less, greater |

### SIMD Performance

**Vectorization** (process multiple u8 values in parallel):

- **SSE (128-bit)**: Process 16× u8 values per instruction
- **AVX2 (256-bit)**: Process 32× u8 values per instruction
- **AVX-512 (512-bit)**: Process 64× u8 values per instruction

```aria
// SIMD-friendly: Compilers can auto-vectorize
void:add_arrays(u8[]:a, u8[]:b, u8[]:result) {
    loop (a.length():i) {
        result[i] = a[i] + b[i];  // Vectorizes to 16/32/64 at a time
    }
}
```

### Memory Efficiency

**Size**: 1 byte per value (smallest integer type)

```aria
// Array memory usage
u8[1_000_000]:data;  // 1 MB (most compact!)

// vs u16
u16[1_000_000]:larger;  // 2 MB (2× size)

// vs u32
u32[1_000_000]:wasteful;  // 4 MB (4× size)

// Cache efficiency: More values per cache line
// 64-byte cache line fits:
// - 64× u8 values (best)
// - 32× u16 values
// - 16× u32 values
```

## Common Gotchas

### 1. Underflow Wrapping (Most Common!)

```aria
u8:count = 10;
count -= 20;  // Wraps to 246 (not -10!)

// Always check before subtraction
if (decrement > count) {
    error("Underflow");
}
```

### 2. Always >= 0 Checks

```aria
u8:value = get_value();
if (value < 0) {  // ALWAYS FALSE!
    // Never executes
}
```

### 3. Multiplication Overflow

```aria
u8:large = 20;
u8:result = large * 20;  // 400 % 256 = 144 (wrapped!)

// Use wider type for intermediate
u16:safe = large.to_u16() * 20;  // 400 (correct)
```

### 4. Mixing with Signed Types

```aria
i8:signed = -1;
u8:unsigned = 10;

bool:comparison = (signed < unsigned);  // FALSE! (-1 → 255)
```

### 5. Division Truncates

```aria
u8:a = 7;
u8:b = 3;
u8:result = a / b;  // 2 (not 2.333...)

// Use f32/f64 for fractional results
f32:quotient = a.to_f32() / b.to_f32();
```

### 6. Char vs u8 Confusion

```aria
// WRONG: u8 is NOT a character
u8:letter = 65;  // This is the NUMBER 65, not 'A'

// CORRECT: Use char for characters
char:letter = 'A';  // This is the CHARACTER 'A'
```

### 7. Loop Decrement Trap

```aria
// WRONG: while (i >= 0) for u8 (always true!)
u8:i = 10;
while (i >= 0) {  // INFINITE!
    i -= 1;
}

// CORRECT: Use different loop structure
loop (10:i) {
    process(i);
}
```

## Summary

**u8** is the **fundamental byte type** for binary data:

**Use u8 when**:
- Binary data (files, networks, memory)
- Image pixels (RGB 0-255, grayscale)
- Small counts and percentages
- Bit manipulation (flags, masks)
- Memory efficiency critical

**Avoid u8 when**:
- Values can be negative (use `i8`)
- Unicode text (use `char`)
- Values >255 (use `u16`, `u32`, `u64`)
- Subtraction might underflow

**Key Principles**:
1. **Never negative** (always >= 0)
2. **Underflow wraps to 255** (0 - 1 = 255!)
3. **Check before subtracting**
4. **Binary data, not text** (use `char` for Unicode)
5. **Perfect for bit manipulation**

**Strengths**:
- ✓ Fundamental binary unit (THE byte)
- ✓ Most compact (1 byte)
- ✓ Best SIMD width (16/32/64 at once)
- ✓ Perfect for images (RGB channels)
- ✓ Ideal for bit manipulation
- ✓ C compatible (`uint8_t`)

**Limitations**:
- ✗ No negative values
- ✗ Underflow wraps silently (0 - 1 = 255)
- ✗ Small range (0-255 only)
- ✗ Not for Unicode text

The `u8` type is the **universal byte** for all binary data in Aria!
