# i8 (8-bit Signed Integer)

The 8-bit signed integer type for values that can be negative.

## Overview

`i8` is an 8-bit signed integer type that can represent values from -128 to +127. It's the smallest signed integer type in Aria, using two's complement representation with wrapping overflow behavior.

**Use `i8` when:**
- You need signed values (can be negative)
- Range of -128 to +127 is sufficient
- Memory efficiency matters (1 byte per value)
- Working with byte-oriented protocols or binary formats
- C interoperability with `int8_t` or `signed char`

**Don't use `i8` when:**
- Values are always positive (use `u8` instead)
- Range might exceed -128 to +127 (use `i16` or `i32`)
- Overflow detection is critical (use TBB types)
- You need symmetric range for `abs()` (use `tbb8`)

**Key characteristics:**
- Range: -128 to +127 (256 total values)
- Size: 1 byte (8 bits)
- Overflow: Wraps around (127 + 1 = -128)
- Underflow: Wraps around (-128 - 1 = 127)
- Sign: Two's complement (MSB is sign bit)
- Performance: Native machine instructions, zero overhead

## i8 vs u8

Choose between signed (`i8`) and unsigned (`u8`) based on whether values can be negative:

```aria
// i8: Can be negative (temperature in Celsius)
temperature: i8 = -40;  // -40°C (valid!)

// u8: Always positive (RGB color channel)
red: u8 = 255;  // 0-255 range (negative makes no sense)
```

**Key differences:**

| Feature | i8 | u8 |
|---------|----|----|
| **Range** | -128 to +127 | 0 to 255 |
| **Can be negative** | ✅ Yes | ❌ No |
| **Maximum positive** | 127 | 255 (2× larger) |
| **Use for** | Offsets, deltas, signed data | Bytes, pixels, counts |
| **Overflow** | 127 + 1 = -128 | 255 + 1 = 0 |

**When to use i8:**
```aria
// Temperature (can be negative)
celsius: i8 = -30;  // ✅ Makes sense

// Coordinate offset (can be negative)
dx: i8 = -5;  // Move 5 pixels left
dy: i8 = 10;  // Move 10 pixels down

// Signed delta
delta: i8 = current - previous;  // Can be positive or negative
```

**When to use u8:**
```aria
// Age (never negative)
age: u8 = 25;  // ✅ u8 is better (0-255 range)

// RGB pixel (0-255)
red: u8 = 255;   // ✅ u8 is correct type
green: u8 = 128;
blue: u8 = 64;

// Byte from file
byte: u8 = file.read_byte();  // ✅ Raw bytes are unsigned
```

**Conversion gotcha:**
```aria
signed: i8 = -1;
unsigned: u8 = signed as u8;  // 255! (bit pattern reinterpreted)

// -1 = 0b11111111 (i8)
//    = 255 (u8, same bit pattern)
```

## i8 vs i16

Choose between `i8` and `i16` based on the range you need:

```aria
// i8: Small range (-128 to +127)
offset: i8 = -50;  // ✅ Fits in i8

// i16: Larger range (-32,768 to +32,767)
altitude: i16 = -1000;  // ❌ Doesn't fit in i8, need i16
```

**Key differences:**

| Feature | i8 | i16 |
|---------|----|----|
| **Range** | -128 to +127 | -32,768 to +32,767 |
| **Size** | 1 byte | 2 bytes (2× larger) |
| **Cache efficiency** | Better (smaller) | Worse (larger) |
| **Use for** | Small signed values | Medium signed values |

**Range boundaries:**
```aria
// i8 boundaries
i8_min: i8 = -128;  // Minimum
i8_max: i8 = 127;   // Maximum

// i16 boundaries
i16_min: i16 = -32768;  // Much larger range
i16_max: i16 = 32767;
```

**When to use i8:**
```aria
// Small temperature difference (-100°C to +100°C fits)
temp_delta: i8 = -15;  // ✅ i8 is enough

// Small coordinate offset
grid_x: i8 = -20;  // ✅ Within -128 to +127
grid_y: i8 = 50;

// Protocol version number (-128 to +127 plenty)
version: i8 = 2;  // ✅ i8 saves memory
```

**When to use i16:**
```aria
// Altitude in meters (can be -1000 to +10000)
altitude: i16 = -500;  // ❌ i8 too small, need i16

// Audio sample (16-bit PCM: -32768 to +32767)
sample: i16 = -15000;  // ❌ i8 can't fit this

// Large coordinate
screen_x: i16 = 1920;  // ❌ Beyond i8 range
```

**Memory efficiency:**
```aria
// 1 million i8 values
small_offsets: [i8; 1000000];  // 1 MB

// 1 million i16 values
large_offsets: [i16; 1000000];  // 2 MB (2× more memory)
```

## i8 vs i32

Choose between `i8` and `i32` based on range and memory needs:

```aria
// i8: Tight range, memory-constrained
flags: i8 = -1;  // ✅ i8 when range guaranteed small

// i32: Default integer, larger range
count: i32 = 1000000;  // ❌ Way beyond i8 range
```

**Key differences:**

| Feature | i8 | i32 |
|---------|----|----|
| **Range** | -128 to +127 | -2.1B to +2.1B |
| **Size** | 1 byte | 4 bytes (4× larger) |
| **Default** | No | ✅ Yes (Aria's default int) |
| **Use for** | Tiny signed values | General-purpose integers |

**When to use i8:**
```aria
// Array of small signed values (save memory)
deltas: [i8; 10000];  // 10 KB (vs 40 KB with i32)

for i in 0..10000 {
    deltas[i] = calculate_small_delta();  // Guaranteed -128 to +127
}

// Binary protocol (exact byte layout)
struct PacketHeader {
    version: i8,     // 1 byte
    packet_type: i8, // 1 byte
    flags: i8,       // 1 byte
    reserved: i8,    // 1 byte
}  // Total: 4 bytes (vs 16 bytes with i32)
```

**When to use i32:**
```aria
// General arithmetic (unknown range)
sum: i32 = 0;
for value in values {
    sum += value;  // ✅ i32 safe for accumulation
}

// Loop counter (beyond i8 range)
for i: i32 in 0..1000 {  // ✅ i8 max is 127, can't fit 1000
    process(i);
}

// Mathematical computation
result: i32 = (x * y) + z;  // ✅ i32 for intermediate calculations
```

**Prefer i32 as default**, use i8 only when you know values fit and need memory efficiency:
```aria
// DEFAULT: Use i32 (Aria's default integer type)
count: i32 = 100;  // ✅ Safe choice

// SPECIAL CASE: Use i8 when range known and memory matters
offsets: [i8; 1000000];  // ✅ Save 3 MB vs i32
```

## Range and Representation

### Range

`i8` can represent 256 distinct values:

- **Minimum:** -128 (-2⁷)
- **Maximum:** +127 (2⁷ - 1)
- **Total values:** 256 (2⁸)
- **Asymmetric:** One more negative value than positive

**Critical asymmetry:**
```aria
// 128 negative values: -128 to -1
// 128 positive values: 0 to +127

// This asymmetry matters for abs()!
min: i8 = -128;
// abs(-128) = 128, but 128 doesn't fit in i8!
// Calling abs(min) PANICS in Aria
```

### Two's Complement Representation

`i8` uses two's complement binary representation:

```aria
// Positive values (MSB = 0)
+127 = 0b01111111 = 0x7F
+1   = 0b00000001 = 0x01
 0   = 0b00000000 = 0x00

// Negative values (MSB = 1)
-1   = 0b11111111 = 0xFF
-128 = 0b10000000 = 0x80
```

**Sign bit (MSB):**
- 0 = positive or zero
- 1 = negative

**Example breakdown:**
```aria
value: i8 = -42;

// Binary: 0b11010110
// Hex:    0xD6
// Sign:   1 (negative, MSB = 1)
// Value:  -42 (two's complement)
```

### Computing Negative Values

To negate a value in two's complement:
1. Flip all bits (bitwise NOT)
2. Add 1

```aria
positive: i8 = 5;  // 0b00000101

// Manual negation:
inverted: i8 = ~positive;  // 0b11111010 (bitwise NOT)
negative: i8 = inverted + 1;  // 0b11111011 = -5

// Or just:
negative: i8 = -positive;  // -5 (compiler does the above)
```

**Verification:**
```aria
value: i8 = -5;  // 0b11111011

// Negate again to get original:
original: i8 = -value;  // 5 (0b00000101)
```

### Boundary Values

```aria
// Minimum value (most negative)
min: i8 = -128;  // 0b10000000 = 0x80

// Maximum value (most positive)
max: i8 = 127;   // 0b01111111 = 0x7F

// Zero
zero: i8 = 0;    // 0b00000000 = 0x00

// -1 (all bits set)
neg_one: i8 = -1;  // 0b11111111 = 0xFF
```

## Overflow and Underflow

### Wrapping Behavior (Defined in Aria)

Unlike C/C++ where signed overflow is undefined behavior, Aria **defines** overflow as two's complement wrapping:

```aria
// Overflow: wraps to minimum
max: i8 = 127;
max += 1;  // max = -128 (wraps around!)

// Underflow: wraps to maximum
min: i8 = -128;
min -= 1;  // min = 127 (wraps around!)
```

**This is defined behavior**, not undefined! Aria guarantees two's complement wrapping.

### Addition Overflow

```aria
a: i8 = 100;
b: i8 = 50;
sum: i8 = a + b;  // 150... but wraps to -106!

// Why?
// 150 = 0b10010110
// MSB = 1, so interpreted as negative
// 0b10010110 = -106 in two's complement
```

**Step-by-step wrapping:**
```aria
value: i8 = 125;
value += 1;  // 126
value += 1;  // 127
value += 1;  // -128 (wrapped!)
value += 1;  // -127
value += 1;  // -126
```

### Subtraction Underflow

```aria
a: i8 = -100;
b: i8 = 50;
diff: i8 = a - b;  // -150... but wraps to 106!

// Why?
// -150 would be 0b01101010 when wrapped
// 0b01101010 = 106 in two's complement
```

**Step-by-step wrapping:**
```aria
value: i8 = -126;
value -= 1;  // -127
value -= 1;  // -128
value -= 1;  // 127 (wrapped!)
value -= 1;  // 126
value -= 1;  // 125
```

### Multiplication Overflow

```aria
a: i8 = 16;
b: i8 = 8;
product: i8 = a * b;  // 128... but wraps to -128!

// Why?
// 128 = 0b10000000
// MSB = 1, so interpreted as -128
```

**More examples:**
```aria
a: i8 = 20;
b: i8 = 7;
product: i8 = a * b;  // 140 wraps to -116

a: i8 = -10;
b: i8 = 20;
product: i8 = a * b;  // -200 wraps to 56
```

### Division (No Overflow)

Division doesn't overflow (quotient always fits), but has special cases:

```aria
a: i8 = 100;
b: i8 = 3;
quotient: i8 = a / b;  // 33 (truncates toward zero)

a: i8 = -100;
b: i8 = 3;
quotient: i8 = a / b;  // -33 (truncates toward zero)
```

**Division by zero:**
```aria
a: i8 = 100;
b: i8 = 0;
quotient: i8 = a / b;  // PANICS! Runtime error
```

### Negation Overflow (Critical!)

```aria
min: i8 = -128;
negated: i8 = -min;  // Should be 128, but 128 doesn't fit!
// Result: -128 (wraps!)

// Why?
// -(-128) = 128, but i8 max is 127
// 128 wraps to -128
```

**This is why `abs(-128)` is dangerous:**
```aria
min: i8 = -128;
absolute: i8 = abs(min);  // PANICS! (No representation for 128)

// abs() is negation for negative values:
// abs(-128) = -(-128) = 128... doesn't fit!
```

## Operations

### Arithmetic Operations

```aria
a: i8 = 10;
b: i8 = 20;

sum: i8 = a + b;       // 30
difference: i8 = a - b;  // -10
product: i8 = a * b;     // -56 (200 wraps!)
quotient: i8 = a / b;    // 0 (truncates toward zero)
remainder: i8 = a % b;   // 10

// Compound assignment
a += 5;   // 15
a -= 3;   // 12
a *= 2;   // 24
a /= 4;   // 6
a %= 5;   // 1
```

### Comparisons

```aria
a: i8 = -10;
b: i8 = 20;

is_equal: bool = a == b;        // false
is_not_equal: bool = a != b;    // true
is_less: bool = a < b;          // true (-10 < 20)
is_less_eq: bool = a <= b;      // true
is_greater: bool = a > b;       // false
is_greater_eq: bool = a >= b;   // false
```

**Signed comparison (respects sign):**
```aria
negative: i8 = -1;
positive: i8 = 127;

result: bool = negative < positive;  // true (signed comparison)

// -1 = 0b11111111
// 127 = 0b01111111
// -1 < 127 in signed comparison
```

### Bitwise Operations

```aria
a: i8 = 0b00110011;  // 51
b: i8 = 0b01010101;  // 85

and_result: i8 = a & b;   // 0b00010001 = 17
or_result: i8 = a | b;    // 0b01110111 = 119
xor_result: i8 = a ^ b;   // 0b01100110 = 102
not_result: i8 = ~a;      // 0b11001100 = -52

// Bit shifts
left: i8 = 1 << 3;   // 0b00001000 = 8
right: i8 = 16 >> 2; // 0b00000100 = 4
```

**Arithmetic right shift (sign extension):**
```aria
negative: i8 = -64;  // 0b11000000

shifted: i8 = negative >> 1;  // 0b11100000 = -32

// MSB propagates (sign extends)
// 0b11000000 >> 1 = 0b11100000 (not 0b01100000)
```

### Negation and Absolute Value

```aria
positive: i8 = 42;
negative: i8 = -positive;  // -42

value: i8 = -30;
abs_value: i8 = abs(value);  // 30
```

**Critical: abs() panics on -128:**
```aria
min: i8 = -128;
abs_min: i8 = abs(min);  // PANICS! (128 doesn't fit in i8)

// Solution 1: Check before calling abs()
if value != -128 {
    abs_value: i8 = abs(value);  // Safe
}

// Solution 2: Use wider type
wide: i16 = value as i16;
abs_wide: i16 = abs(wide);  // abs(-128) = 128 (fits in i16)

// Solution 3: Use TBB type (symmetric range)
safe: tbb8 = value as tbb8;
abs_safe: tbb8 = abs(safe);  // tbb8 min is -127, abs always works
```

### Saturating Arithmetic

Saturating arithmetic clamps results to min/max instead of wrapping:

```aria
fn saturating_add(a: i8, b: i8) -> i8 {
    // Widen to avoid wrapping
    wide: i16 = (a as i16) + (b as i16);
    
    if wide > 127 {
        return 127;  // Saturate at max
    } else if wide < -128 {
        return -128;  // Saturate at min
    } else {
        return wide as i8;
    }
}

a: i8 = 100;
b: i8 = 50;
result: i8 = saturating_add(a, b);  // 127 (saturated, not -106)
```

### Checked Arithmetic

Checked arithmetic detects overflow and returns an error:

```aria
fn checked_add(a: i8, b: i8) -> Result<i8, string> {
    // Overflow if signs match and result sign differs
    sum: i8 = a + b;
    
    if (a > 0 && b > 0 && sum < 0) ||
       (a < 0 && b < 0 && sum > 0) {
        return Err("Overflow");
    }
    
    return Ok(sum);
}

a: i8 = 100;
b: i8 = 50;

match checked_add(a, b) {
    Ok(sum) => println("Sum: {}", sum),
    Err(msg) => println("Error: {}", msg),  // "Error: Overflow"
}
```

**Or use TBB types for automatic checking:**
```aria
a: tbb8 = 100;
b: tbb8 = 50;
sum: tbb8 = a + b;  // ERR (overflow detected automatically)

if sum == ERR {
    println("Overflow detected!");
}
```

## Type Conversions

### Widening (Always Safe)

Converting `i8` to a larger signed type is always safe:

```aria
small: i8 = -100;
medium: i16 = small as i16;  // -100 (exact)
large: i32 = small as i32;   // -100 (exact)
huge: i64 = small as i64;    // -100 (exact)

// Sign extends properly
// -100 (i8) = 0b10011100
// -100 (i16) = 0b1111111110011100 (sign-extended)
```

### Narrowing (Potentially Unsafe)

Converting larger types to `i8` truncates if value doesn't fit:

```aria
large: i32 = 1000;
small: i8 = large as i8;  // -24 (truncated!)

// 1000 = 0b0000001111101000
// Truncate to 8 bits: 0b11101000 = -24 in two's complement
```

**Safe narrowing with range check:**
```aria
fn safe_i32_to_i8(value: i32) -> ?i8 {
    if value < -128 || value > 127 {
        return NIL;  // Out of range
    }
    return value as i8;  // Safe cast
}

large: i32 = 1000;
match safe_i32_to_i8(large) {
    ?small => println("Converted: {}", small),
    NIL => println("Value too large for i8"),
}
```

### Signed to Unsigned

Converting `i8` to `u8` reinterprets the bit pattern:

```aria
signed: i8 = -1;  // 0b11111111
unsigned: u8 = signed as u8;  // 255 (same bit pattern)

signed: i8 = -128;  // 0b10000000
unsigned: u8 = signed as u8;  // 128 (same bit pattern)

signed: i8 = 127;  // 0b01111111
unsigned: u8 = signed as u8;  // 127 (same bit pattern)
```

**Safe conversion (check sign first):**
```aria
fn safe_i8_to_u8(value: i8) -> ?u8 {
    if value < 0 {
        return NIL;  // Negative values don't fit in u8
    }
    return value as u8;  // Safe (0 to 127)
}

signed: i8 = -10;
match safe_i8_to_u8(signed) {
    ?unsigned => println("Converted: {}", unsigned),
    NIL => println("Negative value can't convert to u8"),
}
```

### Unsigned to Signed

Converting `u8` to `i8` reinterprets if value > 127:

```aria
unsigned: u8 = 200;  // 0b11001000
signed: i8 = unsigned as i8;  // -56 (reinterpreted!)

// 200 > 127, so MSB = 1 in i8
// 0b11001000 = -56 in two's complement
```

**Safe conversion:**
```aria
fn safe_u8_to_i8(value: u8) -> ?i8 {
    if value > 127 {
        return NIL;  // Doesn't fit in i8 positive range
    }
    return value as i8;  // Safe (0 to 127)
}

unsigned: u8 = 200;
match safe_u8_to_i8(unsigned) {
    ?signed => println("Converted: {}", signed),
    NIL => println("Value too large for i8"),
}
```

### To Floating-Point

Converting `i8` to floating-point is exact:

```aria
integer: i8 = -42;
float32: f32 = integer as f32;  // -42.0 (exact)
float64: f64 = integer as f64;  // -42.0 (exact)

// All i8 values fit exactly in f32 and f64
// (i8 has 256 values, f32 has 23-bit mantissa)
```

### From Floating-Point

Converting floating-point to `i8` truncates toward zero:

```aria
float: f64 = 42.7;
integer: i8 = float as i8;  // 42 (truncates decimal)

float: f64 = -42.9;
integer: i8 = float as i8;  // -42 (truncates toward zero)
```

**Out of range values wrap or saturate:**
```aria
large: f64 = 1000.0;
small: i8 = large as i8;  // Undefined (implementation-dependent)

// Better: Check range first
fn safe_f64_to_i8(value: f64) -> ?i8 {
    if value < -128.0 || value > 127.0 {
        return NIL;
    }
    return value as i8;
}
```

## Common Patterns

### Temperature in Celsius

```aria
// Temperature sensor reading
temperature: i8 = read_temperature_sensor();

// Valid range: -40°C to +85°C (fits in i8)
if temperature < -40 || temperature > 85 {
    println("Sensor reading out of range!");
}

// Display with sign
if temperature < 0 {
    println("{}°C (below freezing)", temperature);
} else {
    println("{}°C", temperature);
}
```

### Coordinate Offsets

```aria
// Sprite position
x: i32 = 100;
y: i32 = 200;

// Movement deltas (small offsets)
dx: i8 = -5;  // Move 5 pixels left
dy: i8 = 10;  // Move 10 pixels down

// Apply offsets
x += dx as i32;  // 95
y += dy as i32;  // 210

// Relative positioning
fn move_relative(x: i32, y: i32, dx: i8, dy: i8) -> (i32, i32) {
    return (x + dx as i32, y + dy as i32);
}
```

### Signed Delta Encoding

```aria
// Compress data by storing deltas instead of absolute values
fn delta_encode(values: [i32]) -> [i8] {
    deltas: [i8; values.len()];
    
    for i in 1..values.len() {
        delta: i32 = values[i] - values[i - 1];
        
        // Clamp to i8 range
        if delta > 127 {
            deltas[i] = 127;
        } else if delta < -128 {
            deltas[i] = -128;
        } else {
            deltas[i] = delta as i8;
        }
    }
    
    return deltas;
}

// Usage
values: [i32] = [100, 105, 103, 110, 108];
deltas: [i8] = delta_encode(values);  // [0, 5, -2, 7, -2]
```

### Protocol Headers

```aria
struct PacketHeader {
    version: i8,      // Protocol version (-128 to +127)
    packet_type: i8,  // Packet type ID
    flags: i8,        // Bit flags
    priority: i8,     // Priority (-128 = lowest, 127 = highest)
}

fn create_header(packet_type: i8, priority: i8) -> PacketHeader {
    return PacketHeader {
        version: 1,
        packet_type: packet_type,
        flags: 0,
        priority: priority,
    };
}

// Usage
header: PacketHeader = create_header(5, 10);  // Type 5, priority 10
```

### Ring Buffer Index

```aria
struct RingBuffer {
    data: [i8; 128],
    index: i8,  // 0 to 127, wraps naturally
}

fn ring_push(buffer: &mut RingBuffer, value: i8) {
    buffer.data[buffer.index as usize] = value;
    buffer.index += 1;  // Wraps from 127 to -128
    
    // Convert to unsigned for array indexing
    // (or use modulo: buffer.index % 128)
}
```

### Checked Range Conversion

```aria
fn clamp_to_i8(value: i32) -> i8 {
    if value > 127 {
        return 127;
    } else if value < -128 {
        return -128;
    } else {
        return value as i8;
    }
}

// Usage
large_value: i32 = 1000;
clamped: i8 = clamp_to_i8(large_value);  // 127 (saturated)

small_value: i32 = -200;
clamped: i8 = clamp_to_i8(small_value);  // -128 (saturated)
```

### Byte-Oriented Protocol

```aria
fn parse_packet(data: [u8]) -> Result<Packet, string> {
    if data.len() < 4 {
        return Err("Packet too short");
    }
    
    // Parse header (interpret bytes as signed)
    version: i8 = data[0] as i8;
    packet_type: i8 = data[1] as i8;
    flags: i8 = data[2] as i8;
    length: u8 = data[3];  // Unsigned for length
    
    // Validate
    if version < 0 {
        return Err("Invalid version");
    }
    
    return Ok(Packet {
        version: version,
        packet_type: packet_type,
        flags: flags,
        length: length,
    });
}
```

## Anti-Patterns

### ❌ Using i8 When Values Can't Be Negative

```aria
// WRONG: Age is never negative, use u8!
age: i8 = 25;  // ❌ Wastes half the range

// RIGHT: Use u8 for values that can't be negative
age: u8 = 25;  // ✅ 0-255 range, semantically correct
```

### ❌ Decrement Loop That Underflows

```aria
// WRONG: Infinite loop! (wraps at -128)
counter: i8 = 10;
while counter >= 0 {
    process(counter);
    counter -= 1;  // ❌ Wraps from -128 to 127!
}

// RIGHT: Use unsigned type for countdown
counter: u8 = 10;
while counter > 0 {
    counter -= 1;  // ✅ Stops at 0
    process(counter);
}

// OR: Count up instead
for i in 0..10 {
    process(i);  // ✅ Clear loop bounds
}
```

### ❌ Ignoring Overflow in Accumulation

```aria
// WRONG: Overflows silently!
sum: i8 = 0;
for value in values {
    sum += value;  // ❌ Can overflow!
}

// RIGHT: Use larger type or checked arithmetic
sum: i32 = 0;  // ✅ Much larger range
for value in values {
    sum += value as i32;  // Safe accumulation
}

// OR: Use TBB type for automatic checking
sum: tbb16 = 0;
for value in values {
    sum += value as tbb16;
    if sum == ERR {
        println("Overflow detected!");
        break;
    }
}
```

```aria
// WRONG: Panics if value is -128!
value: i8 = read_sensor();

// RIGHT: Check before abs()
value: i8 = read_sensor();
if value == -128 {

// OR: Use wider type

absolute: i16 = abs(wide);  // ✅ abs(-128) = 128 (fits in i16)
```

```aria
// WRONG: Truncates silently!

small: i8 = large as i8;  // ❌ -24 (wrapped!)

// RIGHT: Check range before casting
large: i32 = 1000;
if large >= -128 && large <= 127 {
    small: i8 = large as i8;  // ✅ Safe
} else {
    println("Value doesn't fit in i8");
}

// OR: Use Result type
fn safe_cast(value: i32) -> Result<i8, string> {
    if value < -128 || value > 127 {

    }
    return Ok(value as i8);
```

```aria
// WRONG: Comparison doesn't work as expected!
signed: i8 = -1;   // 0b11111111

// Implicit conversion can cause issues
```

## Performance Characteristics

### Memory Efficiency

`i8` is the smallest integer type:

```aria
// 1 byte per value
value: i8 = 42;  // 1 byte

// Array of 1 million i8
array: [i8; 1000000];  // 1 MB

// Compare to i32:
array32: [i32; 1000000];  // 4 MB (4× larger!)
```

**Cache efficiency:** More values fit in cache lines, improving performance for large arrays.

### Native Instructions

`i8` operations compile to efficient native instructions:

```aria
a: i8 = 10;
b: i8 = 20;
sum: i8 = a + b;
```

**x86-64 assembly:**
```asm
mov al, 10    ; Load 10 into AL register
add al, 20    ; Add 20 to AL
; Result in AL (one instruction!)
```

**Zero overhead:** Same as hand-written assembly.

### SIMD Vectorization

`i8` has excellent SIMD support (process many values at once):

```aria
// Process 16 i8 values simultaneously (SSE)
// Process 32 i8 values simultaneously (AVX2)
// Process 64 i8 values simultaneously (AVX-512)

// Example: Add two arrays
fn add_arrays(a: [i8], b: [i8], result: &mut [i8]) {
    for i in 0..a.len() {
        result[i] = a[i] + b[i];  // Auto-vectorized by compiler
    }
}

// Compiler generates SIMD code for maximum throughput
```

### Register Usage

`i8` operations use 8-bit register parts:

```aria
// x86-64 has 16 general-purpose registers
// Each register can be used as 8-bit, 16-bit, 32-bit, or 64-bit

a: i8 = 10;   // Uses AL (lower 8 bits of RAX)
b: i8 = 20;   // Uses BL (lower 8 bits of RBX)
```

**Efficient:** Modern CPUs handle 8-bit operations natively.

### Comparison to Larger Types

| Type | Size | Cache Lines (64 bytes) | SIMD Width (AVX2) |
|------|------|------------------------|-------------------|
| i8   | 1 byte | 64 values | 32 values |
| i16  | 2 bytes | 32 values | 16 values |
| i32  | 4 bytes | 16 values | 8 values |
| i64  | 8 bytes | 8 values | 4 values |

**i8 advantages:**
- 8× more values per cache line vs i64
- 4× better SIMD throughput vs i64
- 4× less memory bandwidth vs i32

## Common Gotchas

### Gotcha 1: Asymmetric Range (abs Trap)

```aria
// i8 has one more negative value than positive
min: i8 = -128;
max: i8 = 127;

// This means abs() doesn't always work!
absolute: i8 = abs(-128);  // PANICS! (128 doesn't fit)

// ✅ Solution: Check before abs()
if value != -128 {
    absolute: i8 = abs(value);
}

// OR: Use wider type
wide: i16 = value as i16;
absolute: i16 = abs(wide);  // abs(-128) = 128 (fits)
```

### Gotcha 2: Silent Overflow Wrapping

```aria
// Overflow wraps silently (no error!)
value: i8 = 100;
value += 50;  // 150... wraps to -106!

// ✅ Solution: Use checked arithmetic
fn checked_add(a: i8, b: i8) -> ?i8 {
    sum: i8 = a + b;
    
    // Detect overflow
    if (a > 0 && b > 0 && sum < 0) ||
       (a < 0 && b < 0 && sum > 0) {
        return NIL;  // Overflow detected
    }
    
    return sum;
}

// OR: Use TBB types
tbb_value: tbb8 = 100;
tbb_value += 50;  // ERR (automatic overflow detection)
```

### Gotcha 3: Signed vs Unsigned Confusion

```aria
// i8 and u8 have different ranges!
signed: i8 = -1;   // 0b11111111
unsigned: u8 = signed as u8;  // 255 (same bits!)

// Comparisons can be confusing
signed: i8 = -1;
unsigned: u8 = 1;

// Be explicit about conversions
if signed as i16 < unsigned as i16 {  // ✅ Clear
    // Explicit widening
}
```

### Gotcha 4: Narrowing Truncation

```aria
// Narrowing from larger types truncates!
large: i32 = 300;
small: i8 = large as i8;  // 44 (truncated!)

// 300 = 0x0000012C (i32)
// Truncate to 8 bits: 0x2C = 44 (i8)

// ✅ Solution: Check range
if large >= -128 && large <= 127 {
    small: i8 = large as i8;  // Safe
}
```

### Gotcha 5: Division by Zero

```aria
// Division by zero panics!
a: i8 = 100;
b: i8 = 0;
quotient: i8 = a / b;  // PANICS!

// ✅ Solution: Check before dividing
if b != 0 {
    quotient: i8 = a / b;  // Safe
} else {
    // Handle division by zero
}

// OR: Use TBB types
tbb_a: tbb8 = 100;
tbb_b: tbb8 = 0;
tbb_quotient: tbb8 = tbb_a / tbb_b;  // ERR (no panic)
```

### Gotcha 6: Multiplication Overflow

```aria
// Multiplication overflows easily!
a: i8 = 16;
b: i8 = 8;
product: i8 = a * b;  // 128... wraps to -128!

// ✅ Solution: Widen before multiplying
wide_a: i16 = a as i16;
wide_b: i16 = b as i16;
wide_product: i16 = wide_a * wide_b;  // 128 (no overflow)

// Check if fits before narrowing
if wide_product >= -128 && wide_product <= 127 {
    product: i8 = wide_product as i8;
}
```

### Gotcha 7: Arithmetic Right Shift

```aria
// Right shift sign-extends (not logical shift)
negative: i8 = -64;  // 0b11000000

shifted: i8 = negative >> 1;  // 0b11100000 = -32 (not 64!)

// MSB propagates (arithmetic shift)
// To get logical shift, convert to unsigned first
unsigned: u8 = negative as u8;  // 192
logical: u8 = unsigned >> 1;    // 96 (logical shift)
```

### Gotcha 8: Underflow in Loops

```aria
// Countdown loop can underflow!
counter: i8 = 10;
while counter >= 0 {
    process(counter);
    counter -= 1;  // Wraps from -128 to 127!
}

// ✅ Solution: Count up or use unsigned
for i in 0..10 {
    process(9 - i);  // Count down safely
}

// OR: Use unsigned type
counter: u8 = 10;
while counter > 0 {
    counter -= 1;
    process(counter);  // Stops at 0
}
```

## Summary

`i8` is an 8-bit signed integer type for small values that can be negative:

**Key Properties:**
- **Range:** -128 to +127 (256 values)
- **Size:** 1 byte (8 bits)
- **Representation:** Two's complement
- **Overflow:** Wraps around (defined behavior)
- **Performance:** Native instructions, excellent SIMD support

**When to use i8:**
- ✅ Values can be negative (temperature, offsets, deltas)
- ✅ Range guaranteed within -128 to +127
- ✅ Memory efficiency critical (1 byte vs 4 bytes for i32)
- ✅ C interoperability (`int8_t`, `signed char`)
- ✅ Byte-oriented protocols or binary formats

**When NOT to use i8:**
- ❌ Values always positive (use `u8` instead)
- ❌ Range might exceed -128 to +127 (use `i16` or `i32`)
- ❌ Overflow detection critical (use TBB types)
- ❌ Need symmetric range for `abs()` (use `tbb8`)

**Critical gotchas:**
- ⚠️ `abs(-128)` panics (128 doesn't fit)
- ⚠️ Overflow wraps silently (127 + 1 = -128)
- ⚠️ Asymmetric range (one more negative value)
- ⚠️ Narrowing truncates (300 as i8 = 44)
- ⚠️ Mixing with unsigned can confuse (-1 as u8 = 255)

**Best practices:**
- Check range when converting from larger types
- Be aware of overflow wrapping behavior
- Avoid `abs()` on unchecked values (check for -128 first)
- Prefer `u8` when values never negative
- Use TBB types (`tbb8`) for safety-critical code
- Document when wrapping is intentional

**vs Other Types:**

| Comparison | Key Difference | When to Prefer i8 |
|------------|----------------|-------------------|
| **i8 vs u8** | Signed vs unsigned | Values can be negative |
| **i8 vs i16** | 1 byte vs 2 bytes | Values fit in -128 to +127 |
| **i8 vs i32** | 1 byte vs 4 bytes | Memory critical, range small |
| **i8 vs tbb8** | Wrapping vs error detection | Performance > safety |

**Remember:** `i8` is for small signed values where overflow wrapping is acceptable. For safety-critical code with overflow detection, use `tbb8` instead!
