# f64: 64-bit Floating Point

## Overview

**Purpose**: Decimal numbers, scientific calculations, and measurements with high precision.

**Characteristics**:
- 64-bit IEEE 754 double-precision floating point
- Range: approximately ±1.8 × 10^308
- Precision: ~15-16 decimal digits
- Special values: NaN, Infinity, -Infinity, -0.0
- Hardware accelerated (native CPU instructions)
- Default floating-point type in Aria
- Full name: `float64`, common alias: `f64`

**When to Use**:
- **Scientific computing** (physics, chemistry, astronomy)
- **Graphics and rendering** (coordinates, transformations)
- **Statistical analysis** (averages, standard deviations)
- **Measurements** (distances, weights, temperatures)
- **General decimal arithmetic** (non-financial)

**When NOT to Use**:
- **Financial calculations** (use fixed-point types like `fix256` for exact cents)
- **Exact decimal values** (0.1 cannot be represented exactly!)
- **Loop counters** (use `i32` - integers are exact)
- **Equality comparisons** (floating-point rounding makes equality unreliable)
- **Safety-critical systems** (rounding errors can accumulate)

## f64 vs f32: Precision vs Memory

### Size and Precision Comparison

```aria
// f32: 32-bit single precision
f32:small = 3.14159;
// Precision: ~7 decimal digits
// Range: ±3.4 × 10^38
// Memory: 4 bytes

// f64: 64-bit double precision (DEFAULT)
f64:large = 3.14159265358979;
// Precision: ~15-16 decimal digits
// Range: ±1.8 × 10^308
// Memory: 8 bytes
```

### Decision Matrix

| Use Case | Type | Reason |
|----------|------|--------|
| General purpose | `f64` | Default, safer precision |
| Scientific computing | `f64` | Need precision for accuracy |
| 3D graphics positions | `f64` or `f32` | f64 for large worlds, f32 otherwise |
| Colors (0.0-1.0 range) | `f32` | 7 digits sufficient, saves memory |
| Audio samples | `f32` | Humans can't hear the difference |
| Large arrays/GPU | `f32` | 2× more values, 2× faster bandwidth |
| Accumulation | `f64` | Reduce rounding error buildup |
| Intermediate calculations | `f64` | Minimize precision loss |

### Precision Comparison

```aria
// f32 loses precision quickly
f32:pi_32 = 3.14159265358979;
// Actually stores: 3.1415927 (only ~7 digits)

// f64 maintains precision
f64:pi_64 = 3.14159265358979;
// Stores: 3.14159265358979 (~15 digits)

// Large number precision loss in f32
f32:large_32 = 16777217.0;  // 2^24 + 1
// Stores exactly 16777216.0 (loses the +1!)

// f64 still exact
f64:large_64 = 16777217.0;
// Stores exactly 16777217.0
```

## f64 vs i64: Floating vs Integer

### The Critical Distinction

**f64**: Approximate decimals
- Purpose: Real numbers, fractions, scientific notation
- Range: ±1.8 × 10^308 (enormous!)
- Precision: ~15-16 significant digits (NOT exact!)
- Operations: Can lose precision, special values (NaN, Infinity)
- Intent: "This is a measurement or calculation with decimals"

**i64**: Exact integers
- Purpose: Whole numbers, counts, IDs
- Range: ±9.2 quintillion (exact)
- Precision: Every value is exact (no rounding)
- Operations: Wrapping overflow, no special values
- Intent: "This is a countable quantity or exact value"

### Examples of the Difference

```aria
// Floating point: Measurements or decimals
f64:temperature = 98.6;        // Fahrenheit (decimal)
f64:distance_km = 42.195;      // Marathon distance
f64:pi = 3.14159265358979;     // Mathematical constant

// Integer: Counts or exact values
i64:people_count = 7800000000; // World population (exact)
i64:timestamp_ns = 1739318400000000000;  // Exact nanoseconds
i64:array_size = 1000000;      // Exact element count
```

### Large Integer Precision Loss in f64

```aria
// f64 CANNOT represent all i64 values exactly!
i64:exact = 9007199254740993;  // 2^53 + 1
f64:converted = f64(exact);    // Converts but loses precision!
i64:back = i64(converted);     // 9007199254740992 (lost the +1!)

// Safe integer range in f64: -(2^53) to +(2^53)
// Beyond this, not all integers representable
i64:max_safe = 9007199254740992;  // 2^53 (last exactly representable)

// For exact large integers, keep using i64!
```

### When to Use Which

```aria
// Use f64 for:
f64:weight_kg = 72.5;          // Continuous measurement
f64:angle_degrees = 45.0;      // Rotation angle
f64:ratio = 1.618033988749;    // Golden ratio (irrational)

// Use i64 for:
i64:iteration_count = 1000000; // Loop iterations (exact)
i64:user_id = 12345678;        // Database ID (exact)
i64:nanoseconds = 1000000000;  // Time delta (exact)
```

## IEEE 754 Representation

### Bit Layout

```
Bit:  63    62-52         51-0
      Sign  Exponent      Mantissa (Fraction)
      1 bit 11 bits       52 bits
```

**Sign bit** (bit 63):
- `0` = positive
- `1` = negative

**Exponent** (bits 62-52, 11 bits):
- Biased by 1023
- Range: 0 to 2047
- 0: Zero or subnormal
- 1-2046: Normal numbers
- 2047: Infinity or NaN

**Mantissa** (bits 51-0, 52 bits):
- Represents fractional part
- Implicit leading 1 (for normal numbers)
- 52 bits ≈ 15.95 decimal digits precision

### Value Calculation

**Normal numbers**:
```
value = (-1)^sign × 1.mantissa × 2^(exponent - 1023)
```

**Example**: Representing 123.456
```
123.456 = 1.92891... × 2^6

Sign: 0 (positive)
Exponent: 6 + 1023 = 1029
Mantissa: 0.92891... (52 bits)

Stored as: 0x405EDD2F1A9FBE77
```

## Special Values

### Zero

```aria
// Positive zero
f64:pos_zero = 0.0;

// Negative zero (yes, this exists!)
f64:neg_zero = -0.0;

// They compare equal
bool:same = (pos_zero == neg_zero);  // true

// But behave differently in division
f64:pos_inf = 1.0 / pos_zero;  // +Infinity
f64:neg_inf = 1.0 / neg_zero;  // -Infinity

// Check for zero (treats both as zero)
bool:is_zero = (value == 0.0);  // true for both +0.0 and -0.0
```

### Infinity

```aria
// Positive infinity
f64:pos_inf = f64:INFINITY;
f64:pos_inf2 = 1.0 / 0.0;

// Negative infinity
f64:neg_inf = f64:NEG_INFINITY;
f64:neg_inf2 = -1.0 / 0.0;

// Arithmetic with infinity
f64:result1 = pos_inf + 1000.0;      // +Infinity (infinity + finite = infinity)
f64:result2 = pos_inf * 2.0;         // +Infinity
f64:result3 = pos_inf - 1000.0;      // +Infinity
f64:result4 = pos_inf / 2.0;         // +Infinity

// Undefined: infinity - infinity
f64:undefined = pos_inf - pos_inf;   // NaN!

// Check for infinity
bool:is_inf = value.is_infinite();   // true for ±Infinity
bool:is_pos_inf = value.is_positive_infinite();  // true for +Infinity only
```

### NaN (Not a Number)

```aria
// NaN creation
f64:nan1 = f64:NAN;                  // Explicit NaN
f64:nan2 = 0.0 / 0.0;                // 0/0 = NaN
f64:nan3 = f64:INFINITY - f64:INFINITY;  // ∞ - ∞ = NaN
f64:nan4 = (-1.0).sqrt();            // √(-1) = NaN (imaginary!)

// NaN propagates through calculations
f64:result = nan1 + 5.0;             // NaN (NaN + anything = NaN)
f64:result2 = nan1 * 100.0;          // NaN

// ⚠️ CRITICAL: NaN != NaN (NaN is not equal to itself!)
bool:equal = (nan1 == nan2);         // false! (!!!)
bool:equal2 = (nan1 == nan1);        // false! (even comparing to itself!)

// Correct: Check for NaN
bool:is_nan = value.is_nan();        // true if value is NaN

// All comparisons with NaN return false
bool:less = (nan1 < 5.0);            // false
bool:greater = (nan1 > 5.0);         // false
bool:equal = (nan1 == 5.0);          // false
bool:not_equal = (nan1 != 5.0);      // true (only != returns true!)
```

## Arithmetic Operations

### Basic Arithmetic

```aria
f64:a = 10.5;
f64:b = 3.2;

f64:sum = a + b;         // 13.7
f64:difference = a - b;  // 7.3
f64:product = a * b;     // 33.6
f64:quotient = a / b;    // 3.28125
```

### Division by Zero

```aria
f64:pos = 1.0 / 0.0;     // +Infinity (not panic!)
f64:neg = -1.0 / 0.0;    // -Infinity
f64:nan = 0.0 / 0.0;     // NaN (undefined)

// Check before using result
f64:result = a / b;
if (result.is_infinite()) {
    log.error("Division resulted in infinity\n");
} else if (result.is_nan()) {
    log.error("Division resulted in NaN\n");
} else {
    // Use result
}
```

### Precision Loss in Arithmetic

```aria
// Adding small to large loses precision
f64:large = 1.0e16;      // 10 quadrillion
f64:small = 1.0;
f64:result = large + small;  // 10000000000000001.0... maybe?

// Actually: 10000000000000000.0 (lost the +1!)
// Precision is ~15-16 digits, and we're at digit 17

// Subtraction can cause catastrophic cancellation
f64:a = 1.0000000000000001;
f64:b = 1.0;
f64:diff = a - b;  // Should be 1e-16, but rounding errors...
```

## Comparisons

### Equality (Dangerous!)

```aria
// ⚠️ NEVER use == for floating point!
f64:a = 0.1 + 0.2;
f64:b = 0.3;

// This may fail!
if (a == b) {  // Often false! (rounding errors)
    log.write("Equal\n");
}

// CORRECT: Use epsilon comparison
f64:epsilon = 1e-10;  // Tolerance
if (abs(a - b) < epsilon) {
    log.write("Approximately equal\n");
}

// Or use is_close() helper (if available)
if (a.is_close(b, epsilon)) {
    log.write("Close enough\n");
}
```

### Relational Comparisons

```aria
f64:a = 3.14159;
f64:b = 2.71828;

bool:less = (a < b);           // false
bool:less_equal = (a <= b);    // false
bool:greater = (a > b);        // true
bool:greater_equal = (a >= b); // true
```

### NaN in Comparisons

```aria
f64:nan = f64:NAN;
f64:value = 5.0;

// NaN comparisons always return false (except !=)
bool:eq = (nan == value);      // false
bool:lt = (nan < value);       // false
bool:gt = (nan > value);       // false
bool:ne = (nan != value);      // true (only != is true!)

// Even NaN != NaN!
bool:self = (nan == nan);      // false (!!!)

// CORRECT: Use is_nan()
if (value.is_nan()) {
    log.write("Value is NaN\n");
}
```

## Math Functions

### Basic Math

```aria
// Absolute value
f64:abs_val = abs(-42.5);        // 42.5

// Rounding
f64:rounded = round(3.7);        // 4.0
f64:floored = floor(3.7);        // 3.0
f64:ceiled = ceil(3.2);          // 4.0
f64:truncated = trunc(3.7);      // 3.0 (toward zero)

// Min/Max
f64:minimum = min(3.5, 7.2);     // 3.5
f64:maximum = max(3.5, 7.2);     // 7.2
```

### Powers and Roots

```aria
// Square root
f64:sqrt_val = sqrt(16.0);       // 4.0
f64:sqrt_neg = sqrt(-1.0);       // NaN (imaginary number!)

// Power
f64:squared = pow(5.0, 2.0);     // 25.0
f64:cubed = pow(2.0, 3.0);       // 8.0
f64:sqrt_pow = pow(16.0, 0.5);   // 4.0 (same as sqrt)

// Exponential and logarithm
f64:exp_val = exp(1.0);          // 2.71828... (e)
f64:log_nat = ln(2.71828);       // ≈1.0 (natural log)
f64:log_10 = log10(100.0);       // 2.0
f64:log_2 = log2(8.0);           // 3.0
```

### Trigonometry

```aria
// Angles in radians
f64:pi = 3.14159265358979;

// Basic trig
f64:sin_val = sin(pi / 2.0);     // 1.0 (sin(90°))
f64:cos_val = cos(pi);           // -1.0 (cos(180°))

f64:acos_val = acos(-1.0);       // π
f64:atan_val = atan(1.0);        // π/4

// atan2: handles quadrants correctly
f64:angle = atan2(y, x);         // Angle from origin to (x, y)
```

### Hyperbolic Functions

```aria

f64:cosh_val = cosh(0.0);        // 1.0
f64:tanh_val = tanh(0.0);        // 0.0
```

## Type Conversions

### Integer to Float

```aria
// i32 → f64 (always exact for i32 values)
i32:int_val = 42;
f64:float_val = f64(int_val);    // 42.0 (exact)

// i64 → f64 (may lose precision for large values!)
i64:large = 9007199254740993;    // 2^53 + 1
f64:converted = f64(large);      // Loses the +1! (rounds to 2^53)

// Safe range: -(2^53) to +(2^53)
// Beyond this, not all integers representable exactly
```

### Float to Integer

```aria
// f64 → i64 (truncates decimal part)
f64:decimal = 3.14159;
i64:truncated = i64(decimal);    // 3 (decimal part lost)

f64:negative = -2.7;
i64:neg_truncated = i64(negative);  // -2 (truncates toward zero)

// Overflow if f64 value exceeds i64 range
f64:huge = 1.0e100;
// i64:overflow = i64(huge);  // Undefined! (exceeds i64 range)

// Safe: Check range first
f64:value = get_input();
if (value >= f64(i64:MIN) && value <= f64(i64:MAX)) {
    i64:safe = i64(value);
} else {
    log.error("Value out of i64 range\n");
}
```

### String Conversions

```aria
// f64 → string
f64:pi = 3.14159265358979;
string:str = pi.to_string();     // "3.14159265358979"

// With precision control
string:short = pi.to_string(2);  // "3.14"
string:long = pi.to_string(10);  // "3.1415926536"

// Scientific notation
f64:large = 6.02214076e23;
string:scientific = large.to_string_scientific();  // "6.02214076e+23"

// string → f64
string:input = "3.14159";
Result<f64>:parsed = input.parse_f64();

if (parsed.is_ok()) {
    f64:value = parsed.unwrap();  // 3.14159
} else {
    log.error("Failed to parse\n");
}
```

## Common Patterns

### Averaging

```aria
// Calculate average of array
f64:values[5] = [1.5, 2.3, 3.1, 4.7, 5.9];

f64:sum = 0.0;
for (value in values) {
    sum += value;
}

f64:average = sum / f64(values.length());  // 3.5
```

### Distance Calculation

```aria
// Euclidean distance between two points
struct Point {
    f64:x;
    f64:y;
}

func distance(p1: Point, p2: Point): f64 {
    f64:dx = p2.x - p1.x;
    f64:dy = p2.y - p1.y;
    return sqrt(dx * dx + dy * dy);
}

Point:a = {x: 0.0, y: 0.0};
Point:b = {x: 3.0, y: 4.0};
f64:dist = distance(a, b);  // 5.0 (3-4-5 triangle)
```

### Linear Interpolation

```aria
// Lerp: blend between two values
func lerp(a: f64, b: f64, t: f64): f64 {
    return a + (b - a) * t;
}

f64:start = 0.0;
f64:end = 100.0;
f64:halfway = lerp(start, end, 0.5);  // 50.0
f64:quarter = lerp(start, end, 0.25); // 25.0
```

### Clamping

```aria
// Clamp value to range [min, max]
func clamp(value: f64, min_val: f64, max_val: f64): f64 {
    if (value < min_val) return min_val;
    if (value > max_val) return max_val;
    return value;
}

f64:clamped1 = clamp(150.0, 0.0, 100.0);  // 100.0 (capped at max)
f64:clamped2 = clamp(-50.0, 0.0, 100.0);  // 0.0 (capped at min)
f64:clamped3 = clamp(50.0, 0.0, 100.0);   // 50.0 (within range)
```

## Anti-Patterns

### ❌ Using f64 for Money

```aria
// WRONG: Floating point for currency
f64:price = 19.99;
f64:quantity = 3.0;
f64:total = price * quantity;  // 59.97... or 59.96999999999?

f64:balance = 100.0;
balance -= 0.1;  // 99.9... or 99.89999999999?

// After many transactions, accumulates rounding errors!

// RIGHT: Use fixed-point (cents as integers or fix256)
i64:price_cents = 1999;  // $19.99 as cents
i64:quantity = 3;
i64:total_cents = price_cents * quantity;  // 5997 cents = $59.97 (exact!)

// Or use fix256 for exact decimal arithmetic
fix256:price = fix256.from_decimal("19.99");
fix256:total = price * fix256.from_decimal("3");
```

### ❌ Equality Comparisons

```aria
// WRONG: Direct equality
f64:a = 0.1 + 0.2;
f64:b = 0.3;

if (a == b) {  // Often fails! (0.30000000000000004 != 0.3)
    do_something();
}

// RIGHT: Epsilon comparison
f64:epsilon = 1e-10;
if (abs(a - b) < epsilon) {
    do_something();
}
```

### ❌ Loop Counters

```aria
// WRONG: Floating point loop counter
for (f64:i = 0.0; i < 10.0; i += 0.1) {  // Accumulates rounding errors!
    process(i);
}
// Might run 99 or 101 times instead of 100!

// RIGHT: Integer loop counter
for (i32:i = 0; i < 100; i += 1) {
    f64:value = f64(i) * 0.1;  // Convert to f64 inside loop
    process(value);
}
```

### ❌ Ignoring NaN

```aria
// WRONG: Not checking for NaN
f64:result = sqrt(user_input);
// If user_input < 0, result is NaN!

f64:calculation = result * 100.0;  // NaN propagates!
// Now calculation is NaN too, silently corrupting data

// RIGHT: Check for NaN
f64:result = sqrt(user_input);
if (result.is_nan()) {
    log.error("Invalid input (negative square root)\n");
    return;
}
```

### ❌ Assuming Associativity

```aria
// WRONG: Assuming (a + b) + c == a + (b + c)
f64:a = 1.0e20;
f64:b = 1.0;
f64:c = -1.0e20;

f64:result1 = (a + b) + c;  // (1e20 + 1) + (-1e20) ≈ 0 (loses the 1)
f64:result2 = a + (b + c);  // 1e20 + (1 + (-1e20)) = 1e20 + 1 ≈ 1e20

// result1 != result2 due to precision limits!

// Floating-point arithmetic is NOT associative!
```

## Best Practices

### ✅ DO: Use f64 for General Measurements

```aria
// Scientific and engineering calculations
f64:temperature_c = 23.5;
f64:distance_m = 1234.567;
f64:weight_kg = 72.3;
```

### ✅ DO: Use Epsilon for Comparisons

```aria
// Always use tolerance for equality
const f64:EPSILON = 1e-10;

func approx_equal(a: f64, b: f64): bool {
    return abs(a - b) < EPSILON;
}

if (approx_equal(calculated, expected)) {
    // Close enough!
}
```

### ✅ DO: Check for Special Values

```aria
f64:result = do_calculation();

if (result.is_nan()) {
    log.error("Result is NaN\n");
} else if (result.is_infinite()) {
    log.error("Result is infinite\n");
} else {
    // Safe to use
}
```

### ✅ DO: Use Integer Types for Exact Values

```aria
// Use i32/i64 for counts and IDs
i32:count = 1000;  // Exact

// Use f64 only when decimals needed
f64:average = f64(sum) / f64(count);
```

### ❌ DON'T: Use f64 for Financial Math

```aria
// WRONG
f64:balance = 1000.50;

// RIGHT
i64:balance_cents = 100050;  // Store as cents
// Or use fix256 for exact decimal
```

### ❌ DON'T: Compare f64 with ==

```aria
// WRONG
if (value == 0.0) { }

// RIGHT
if (abs(value) < 1e-10) { }
```

### ❌ DON'T: Use f64 for Loop Counters

```aria
// WRONG
for (f64:i = 0.0; i < 1.0; i += 0.01) { }

// RIGHT
for (i32:i = 0; i < 100; i += 1) {
    f64:value = f64(i) * 0.01;
}
```

## Performance Characteristics

### Hardware Acceleration

```aria
// f64 arithmetic is native CPU instruction (very fast)
f64:a = 3.14159;
f64:b = 2.71828;
f64:sum = a + b;  // Single CPU instruction (addsd on x86-64)
```

**Modern CPUs**:
- Addition, subtraction, multiplication: 1-4 cycles latency
- Division: ~10-20 cycles
- Square root: ~10-20 cycles
- Transcendental functions (sin, cos, etc.): ~50-200 cycles

### Memory Bandwidth

```aria
// f64 uses 8 bytes
f64:values[1000000];
// Memory: 8 MB

// f32 uses 4 bytes (half the memory)
f32:values_32[1000000];
// Memory: 4 MB

// Trade-off: f32 saves memory, f64 has better precision
```

### Cache Efficiency

- Cache lines are 64 bytes
- 8 f64 values per cache line
- 16 f32 values per cache line (2× better cache utilization)

## Common Gotchas

### 1. 0.1 + 0.2 ≠ 0.3

```aria
f64:sum = 0.1 + 0.2;
// sum = 0.30000000000000004 (NOT exactly 0.3!)

// Binary cannot represent 0.1 or 0.2 exactly
// Use epsilon comparisons!
```

### 2. NaN != NaN

```aria
f64:nan = f64:NAN;
bool:equal = (nan == nan);  // false! (always!)

// Must use is_nan()
if (value.is_nan()) { }
```

### 3. Large + Small Loses Small

```aria
f64:large = 1.0e16;
f64:small = 1.0;
f64:sum = large + small;  // 1.0e16 (lost the +1!)

// Precision is ~16 digits, and large is already 17 digits
```

### 4. Division by Zero Doesn't Panic

```aria
f64:result = 1.0 / 0.0;  // Infinity (NO panic!)

// Must check result
if (result.is_infinite()) {
    log.error("Division by zero\n");
}
```

### 5. Negative Zero Exists

```aria
f64:neg_zero = -0.0;
f64:pos_zero = 0.0;

bool:equal = (neg_zero == pos_zero);  // true

// But different in 1/x:
f64:pos_inf = 1.0 / pos_zero;  // +Infinity
f64:neg_inf = 1.0 / neg_zero;  // -Infinity
```
