# Async Runtime Executor Implementation

## Status: ✅ COMPLETE

## Overview

The Aria async runtime provides a complete execution environment for coroutines generated by the compiler's async/await system. It includes an executor, task scheduler, future management, and C API for integration with LLVM-generated code.

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                   Aria Generated Code                        │
│  async func:foo = int64() { await bar(); }                  │
└────────────────┬────────────────────────────────────────────┘
                 │ LLVM IR (i8* coroutine handles)
                 ↓
┌─────────────────────────────────────────────────────────────┐
│                    C Runtime API                             │
│  aria_executor_spawn(), aria_executor_run()                  │
│  aria_future_create(), aria_coro_resume()                    │
└────────────────┬────────────────────────────────────────────┘
                 │
                 ↓
┌─────────────────────────────────────────────────────────────┐
│                 C++ Runtime Implementation                    │
│  ┌─────────────┐  ┌──────────┐  ┌──────────────┐           │
│  │  Executor   │  │  Future  │  │  Coroutine   │           │
│  │             │  │          │  │  Intrinsics  │           │
│  │ - Task queue│  │ - State  │  │              │           │
│  │ - Scheduler │  │ - Value  │  │ - Resume     │           │
│  │ - Run loop  │  │ - Poll   │  │ - Done       │           │
│  └─────────────┘  └──────────┘  └──────────────┘           │
└─────────────────────────────────────────────────────────────┘
```

## Components

### 1. Executor (`executor.h`, `executor.cpp`)

**Purpose**: Manages and schedules async tasks.

**Key Classes**:
- `Executor` - Single-threaded task executor
- `Task` - Represents an async task (coroutine instance)
- `TaskState` - Task lifecycle states (PENDING, RUNNING, SUSPENDED, READY, COMPLETED, FAILED)

**Execution Model**:
- **Run-to-completion**: Tasks execute cooperatively until completion
- **Single-threaded**: No preemption, tasks yield at `await` points
- **Ready queue**: FIFO queue of tasks ready to execute

**API**:
```cpp
Executor exec;
TaskId id = exec.spawn(coro_handle);  // Add task to ready queue
exec.runToCompletion();                // Execute until all tasks done
bool hasMore = exec.step();            // Execute one task
```

### 2. Future (`future.h`)

**Purpose**: Represents the result of an async computation.

**Key Classes**:
- `Future` - Generic future for type T (type-erased with void*)
- `FutureState` - PENDING, READY, ERROR
- `PollResult` - Result of polling: PENDING or READY

**Features**:
- Type-erased storage (void* for T)
- Move-only semantics
- Poll-based readiness checking
- Error propagation

**API**:
```cpp
Future fut(sizeof(int64_t));
fut.setValue(&value, sizeof(value));
if (fut.poll() == PollResult::READY) {
    void* result = fut.getValue();
}
```

### 3. Coroutine Intrinsics (`coroutine.h`, `coroutine.cpp`)

**Purpose**: Wrappers for LLVM coroutine intrinsics.

**Functions**:
```c
void __aria_coro_resume(void* handle);   // Resume suspended coroutine
bool __aria_coro_done(void* handle);     // Check if coroutine completed
void __aria_coro_destroy(void* handle);  // Destroy coroutine frame
```

**Note**: These are stubs for testing. In production, LLVM's coroutine lowering pass provides the real implementations.

### 4. C Runtime API (`runtime_api.h`, `runtime_api.cpp`)

**Purpose**: FFI layer between LLVM-generated code and C++ runtime.

**Executor API**:
```c
AriaExecutorHandle aria_get_global_executor(void);
AriaExecutorHandle aria_executor_create(void);
void aria_executor_destroy(AriaExecutorHandle);
AriaTaskId aria_executor_spawn(AriaExecutorHandle, AriaCoroutineHandle);
void aria_executor_run(AriaExecutorHandle);
bool aria_executor_step(AriaExecutorHandle);
```

**Future API**:
```c
AriaFutureHandle aria_future_create(size_t type_size);
void aria_future_destroy(AriaFutureHandle);
bool aria_future_poll(AriaFutureHandle);
bool aria_future_is_ready(AriaFutureHandle);
void* aria_future_get_value(AriaFutureHandle);
void aria_future_set_value(AriaFutureHandle, const void*, size_t);
```

**Memory API**:
```c
void* aria_coro_alloc(size_t size);  // Allocate coroutine frame
void aria_coro_free(void* ptr);      // Free coroutine frame
```

**Helper API**:
```c
void* aria_await_sync(AriaCoroutineHandle);  // Synchronous await
```

## Integration with Compiler

### Generated Code Example

**Aria Source**:
```aria
async func:compute = int64(int64:x) {
    return x * 2;
};
```

**Generated LLVM IR** (simplified):
```llvm
define ptr @compute(i64 %x) {
entry:
  %coro.id = call token @llvm.coro.id(...)
  %coro.size = call i64 @llvm.coro.size.i64()
  %coro.alloc = call ptr @malloc(i64 %coro.size)
  %coro.handle = call ptr @llvm.coro.begin(token %coro.id, ptr %coro.alloc)
  
  ; Function body
  %result = mul i64 %x, 2
  
  ; Cleanup
  %done = call i1 @llvm.coro.end(ptr %coro.handle, i1 false)
  ret ptr %coro.handle
}
```

**Runtime Usage**:
```cpp
// Spawn the async task
void* coro = compute(21);
AriaTaskId task = aria_executor_spawn(NULL, coro);  // NULL = global executor

// Run to completion
aria_executor_run(NULL);

// Result is in coroutine frame
```

## Task Lifecycle

```
     spawn()
        ↓
    ┌───────┐
    │PENDING│
    └───┬───┘
        │ added to ready queue
        ↓
    ┌───────┐
    │ READY │ ←──────────┐
    └───┬───┘            │
        │ step()         │
        ↓                │
    ┌───────┐            │
    │RUNNING│            │
    └───┬───┘            │
        │                │
        ├─► await ──────►│ SUSPENDED
        │                │
        ├─► complete     │
        ↓                │
    ┌───────────┐        │
    │ COMPLETED │        │
    └───────────┘        │
        or               │
    ┌───────┐            │
    │FAILED │            │
    └───────┘            │
                         │
    dependency ready ────┘
```

## Current Limitations

1. **Single-threaded only**: No work-stealing or thread pool
2. **No async I/O**: File/network operations still blocking
3. **No dependency tracking**: Suspended tasks must be manually resumed
4. **Basic scheduling**: FIFO queue, no priorities
5. **malloc/free memory**: Not integrated with Aria's GC yet

## Future Enhancements

1. **Multi-threaded executor**: Work-stealing scheduler
2. **Async I/O runtime**: epoll/kqueue integration
3. **Smart dependency tracking**: Auto-resume dependent tasks
4. **Priority scheduling**: High/low priority task queues
5. **GC integration**: Coroutine frames managed by GC
6. **Async streams**: Iterator-based async sequences
7. **Combinators**: join(), race(), timeout(), select()
