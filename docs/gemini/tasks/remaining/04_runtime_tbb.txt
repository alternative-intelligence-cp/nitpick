**AUDIT FOCUS: Runtime Safety - TBB (Twisted Balanced Binary) Operations**

**Threat Model:**
TBB provides symmetric signed integers with sticky error sentinels. A bug in widening operations could cause: int16→int32 conversion loses error flag → downstream code thinks value is valid → uses invalid sensor reading → robot calculates wrong obstacle position → collision with child. Error propagation is THE critical safety feature.

**Target Files:**
- `src/runtime/tbb/tbb.cpp`

**Specific Items to Audit (5 items max):**

1. **Widening operations (tbb_widen_i16_to_i32, tbb_widen_i32_to_i64, etc.)**
   - Verify sign extension preserves value correctly
   - Check error sentinel (-1 in high bits) detected in source
   - Confirm sentinel propagated to wider type's sentinel value

2. **Narrowing operations (tbb_narrow_i64_to_i32, etc.)**
   - Verify value-out-of-range returns sentinel in target type
   - Check sign preservation during truncation
   - Confirm no silent overflow/underflow

3. **Sentinel detection (tbb_is_sentinel_i32, tbb_is_sentinel_i64, etc.)**
   - Verify correct sentinel value checked for each type
   - Check no false positives (valid negative numbers misidentified)
   - Confirm consistent across all integer sizes

4. **Error propagation through arithmetic**
   - Verify sentinel input → sentinel output for all ops
   - Check operations never "heal" sentinel values
   - Confirm binary ops with one sentinel input → sentinel output

5. **TBB symmetry properties**
   - Verify sign representation is truly symmetric
   - Check -0 does not exist (no trap representation)
   - Confirm absolute value of min equals absolute value of max

**SIL-4 Compliance Requirements:**
- Sentinels MUST be sticky (once error, always error)
- Widening MUST preserve sentinels (no information loss)
- Narrowing MUST detect out-of-range values
- No value can escape error state
- Sentinel detection must be 100% reliable (no false negatives)
