**AUDIT FOCUS: Integration - System Call Wrappers (io_uring)**

**Threat Model:**
System calls are trust boundary to kernel. io_uring provides async I/O but has complex interface. Bug: SQE (submission queue entry) corrupted → kernel executes wrong syscall → opens /dev/motor instead of /dev/sensor → robot reads motor encoder as sensor → garbage data → wrong control decisions.

**Target Files:**
- `src/runtime/io/uring.cpp` (if exists - io_uring wrapper)
- `src/runtime/io/syscall.cpp` (system call wrappers)

**Specific Items to Audit (5 items max):**

1. **SQE (Submission Queue Entry) construction**
   - Verify SQE fields set correctly for each syscall type
   - Check opcode matches intended operation
   - Confirm file descriptors validated before use

2. **CQE (Completion Queue Entry) handling**
   - Verify return codes checked for errors
   - Check result values validated before use
   - Confirm no assumption that syscall succeeded

3. **Buffer registration safety**
   - Verify buffers pinned before registration
   - Check buffer lifetime extends through I/O completion
   - Confirm no use-after-free of I/O buffers

4. **File descriptor lifecycle**
   - Verify file descriptors validated (>= 0)
   - Check descriptors closed when done
   - Confirm no use-after-close

5. **io_uring queue bounds checking**
   - Verify submission queue not overfilled
   - Check completion queue drained regularly
   - Confirm no queue overflow or underflow

**SIL-4 Compliance Requirements:**
- All syscall parameters MUST be validated
- Return codes MUST be checked (no ignored errors)
- File descriptors MUST be managed correctly
- Buffers MUST have correct lifetime
- Queue invariants MUST be maintained
