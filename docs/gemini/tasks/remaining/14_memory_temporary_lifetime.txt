**AUDIT FOCUS: Memory Safety - Temporary Value Lifetime**

**Threat Model:**
Temporary values (expression results) have limited lifetime. Bug: reference to temporary outlives the temporary → dangling reference → reads freed memory → garbage data used in calculation → wrong sensor fusion result → robot thinks obstacle is 2m away when it's 0.5m → collision.

**Target Files:**
- `src/backend/ir/codegen_expr.cpp` (temporary value handling)
- `src/semantic/lifetime_analysis.cpp` (if exists - lifetime tracking)

**Specific Items to Audit (5 items max):**

1. **Temporary value storage**
   - Verify temporaries allocated in appropriate scope
   - Check temporary lifetime matches expression evaluation
   - Confirm no temporary values freed while still referenced

2. **Reference to temporary detection**
   - Verify compiler detects references that outlive temporaries
   - Check error message clear about lifetime issue
   - Confirm no silent acceptance of dangling references

3. **Temporary value propagation in function calls**
   - Verify temporary passed to function lives for call duration
   - Check return value temporaries handled correctly
   - Confirm no use-after-free when returning references

4. **Temporary arrays and slices**
   - Verify array literal lifetime matches usage scope
   - Check slice of temporary array doesn't outlive array
   - Confirm no dangling pointers to expired temporaries

5. **Complex expression temporaries (nested calls, etc.)**
   - Verify intermediate results preserved until expression complete
   - Check no premature deallocation in expression tree
   - Confirm evaluation order guarantees lifetime safety

**SIL-4 Compliance Requirements:**
- References to temporaries MUST be rejected at compile time
- Temporary lifetime MUST be conservatively estimated
- All temporary values MUST be freed (no leaks)
- No use-after-free of temporaries under any code pattern
- Lifetime errors must be compile-time (not runtime crashes)

**Expected Deliverable:**
For each item, report:
- ✅ Compliant / ❌ Bug found / ⚠️ Unclear (need clarification)
- If bug: Exact line number, root cause, proposed fix
- If compliant: Briefly confirm what makes it safe
- If unclear: Specific question about spec or implementation

**Critical Note:**
Lifetime bugs are among the hardest to detect. They often work "most of the time" and only fail when specific code patterns combine. Check interactions between features carefully.
