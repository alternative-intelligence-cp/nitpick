**AUDIT FOCUS: Runtime Safety - LBIM Basic Operations (int128/int256/int512)**

**Threat Model:**

Example: int256 overflow in torque calculation → 1000N becomes -1000N → robot crushes child's hand instead of gently holding it. These operations are used constantly in physics simulation loops (60Hz+), so even rare edge cases WILL occur.

**Target Files:**
- `src/runtime/math/lbim.cpp`
