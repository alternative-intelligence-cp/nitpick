**AUDIT FOCUS: Integration - Foreign Function Call ABI**

**Threat Model:**
FFI calls cross ABI boundary between Aria and C. ABI mismatch: Aria passes struct by value, C expects pointer → stack corruption → return address overwritten → control flow hijacked → arbitrary code execution → robot reprogrammed maliciously.

**Target Files:**
- `src/backend/ir/codegen.cpp` (FFI calling convention)
- `src/backend/ir/ffi_abi.cpp` (if exists - ABI handling)

**Specific Items to Audit (5 items max):**

1. **Calling convention correctness**
   - Verify C functions called with correct convention (cdecl, etc.)
   - Check register usage matches platform ABI
   - Confirm stack alignment requirements met

2. **Struct passing ABI**
   - Verify small structs passed correctly (register vs stack)
   - Check large structs passed by pointer
   - Confirm struct layout matches C compiler expectations

3. **Variadic function handling**
   - Verify variadic args passed correctly (printf, etc.)
   - Check va_list handling matches platform
   - Confirm type safety maintained for varargs

4. **Return value handling**
   - Verify return values extracted from correct location
   - Check large return values handled via hidden pointer
   - Confirm no register corruption on return

5. **Platform-specific ABI quirks**
   - Verify x86_64 SysV ABI rules followed on Linux
   - Check ARM AAPCS compliance if targeting ARM
   - Confirm no assumptions about register sizes or endianness

**SIL-4 Compliance Requirements:**
- Calling conventions MUST match platform ABI exactly
- Struct passing MUST follow ABI rules
- Return values MUST be extracted correctly
- Stack alignment MUST be maintained
- No undefined behavior from ABI violation

**Expected Deliverable:**
For each item, report:
- ✅ Compliant / ❌ Bug found / ⚠️ Unclear (need clarification)
- If bug: Exact line number, root cause, proposed fix
- If compliant: Briefly confirm what makes it safe
- If unclear: Specific question about spec or implementation

**Critical Note:**
ABI bugs are subtle and platform-specific. They may work on one compiler/OS but fail on another. Check against ABI specifications, not just observed behavior.
