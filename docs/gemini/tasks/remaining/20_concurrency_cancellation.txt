**AUDIT FOCUS: Concurrency - Task Cancellation Safety**

**Threat Model:**
Cancellation allows stopping async tasks. Bug: task cancelled mid-operation → cleanup doesn't run → motor still enabled → task thinks it stopped but motor keeps running → uncontrolled movement. Cancellation must guarantee cleanup.

**Target Files:**
- `src/runtime/async/task.cpp` (task cancellation)
- `src/runtime/async/executor.cpp` (task executor)

**Specific Items to Audit (5 items max):**

1. **Cancellation point identification**
   - Verify cancellation checked at safe points only
   - Check no cancellation mid-critical-section
   - Confirm cancellation points clearly marked in code

2. **Cleanup on cancellation**
   - Verify defer blocks run when task cancelled
   - Check resources freed on cancellation
   - Confirm no leaks from cancelled tasks

3. **Cancellation propagation in nested tasks**
   - Verify parent cancellation cancels children
   - Check child cancellation doesn't cancel parent
   - Confirm cancellation scope clearly defined

4. **Partial operation handling**
   - Verify partial writes rolled back on cancellation
   - Check invariants maintained even if cancelled
   - Confirm no observable partial state after cancel

5. **Race between completion and cancellation**
   - Verify task completion prevents cancellation
   - Check cancellation doesn't affect completed task result
   - Confirm no double-cleanup (both completion and cancellation)

**SIL-4 Compliance Requirements:**
- Cancellation MUST run cleanup (defer, etc.)
- Cancelled tasks MUST NOT leave partial state
- Cancellation points MUST be safe
- Resources MUST be freed on cancellation
- No race conditions between cancel and complete

**Expected Deliverable:**
For each item, report:
- ✅ Compliant / ❌ Bug found / ⚠️ Unclear (need clarification)
- If bug: Exact line number, root cause, proposed fix
- If compliant: Briefly confirm what makes it safe
- If unclear: Specific question about spec or implementation

**Critical Note:**
Cancellation is similar to panic - must guarantee cleanup. But unlike panic, cancellation is normal operation, so must be efficient and race-free.
