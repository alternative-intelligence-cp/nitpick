**AUDIT FOCUS: Type System - Generic Type Resolution and Monomorphization**

**Threat Model:**
Generic types require monomorphization (creating concrete versions for each type parameter combination). Bugs here can cause: wrong type substituted → struct size mismatch → memory corruption. Example: `Vec<i32>` accidentally instantiated as `Vec<i16>` → allocates 2 bytes per element instead of 4 → buffer overflow when storing i32 values.

**Target Files:**
- `src/semantic/type_checker.cpp` (generic type instantiation)
- `src/backend/ir/codegen.cpp` (monomorphization)

**Specific Items to Audit (5 items max):**

1. **Type parameter substitution correctness**
   - Verify all occurrences of type parameter replaced with concrete type
   - Check no missed substitutions in nested contexts
   - Confirm consistent substitution across entire generic definition

2. **Generic function instantiation**
   - Verify separate LLVM function generated for each type combination
   - Check function name mangling prevents collisions
   - Confirm each instantiation has correct type-specific code

3. **Generic struct instantiation**
   - Verify struct layout computed correctly for each type parameter
   - Check field offsets match concrete type sizes
   - Confirm no struct layout shared between incompatible instantiations

4. **Type parameter constraints (bounds checking)**
   - Verify type parameters satisfy all declared constraints
   - Check constraint violations cause compile error (not runtime error)
   - Confirm constraints checked before monomorphization

5. **Recursion in generic types (e.g., `struct Node<T> { next: Optional<Node<T>> }`)**
   - Verify recursive instantiation terminates correctly
   - Check no infinite monomorphization loops
   - Confirm indirection (via pointer/optional) breaks recursion properly

**SIL-4 Compliance Requirements:**
- Type substitution MUST be complete and consistent
- Each monomorphization MUST be type-safe independently
- No code sharing between incompatible instantiations
- Generic constraints MUST be enforced before instantiation
- Recursive types must be structurally sound (finite size)

**Critical Note:**
Generic type bugs are subtle because they may only appear with specific type parameter combinations. Test with various types, especially edge cases (zero-sized types, very large types).
