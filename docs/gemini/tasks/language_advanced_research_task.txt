ARIA LANGUAGE RESEARCH: Concurrency, FFI, Standard Library & Compilation Model

You are an expert programming language designer with deep knowledge of concurrency models, foreign function interfaces, standard library design, and compilation systems. You are helping design the Aria programming language.

CONTEXT:
- Aria is a modern systems language with LLVM backend
- Has async runtime (M:N threading), must interface with C (especially for Nikola consciousness substrate)
- Compiles to native code, supports separate compilation and package system
- See attached files for compiler architecture, runtime library, and ecosystem overview

YOUR TASK:
Provide comprehensive research and design recommendations for the following 7 topics. For each topic, analyze trade-offs, provide detailed implementation strategies, and give concrete code examples.

TOPICS:

1. ASYNC/AWAIT SYNTAX & SEMANTICS
   - Async function declaration: `async func:fetch = result<string>(url: string) { ... }`
   - Await syntax: `data = await fetch("https://...");`
   - How to represent Future<T> type in type system?
   - Async functions return Future<result<T>>? Or result<Future<T>>? Or something else?
   - Integration with Result<T>: Error handling in async context
   - Async and TBB: Can async function return tbb8? How does ERR propagate through await?
   - Blocking semantics: What happens when await blocks?

2. FUTURE/PROMISE IMPLEMENTATION - LANGUAGE TO RUNTIME MAPPING
   - How does `async func` compile to LLVM IR + runtime calls?
   - Future<T> representation: Heap-allocated state machine?
   - await lowering: Suspend/resume mechanics
   - Task scheduler integration: How to enqueue async tasks on M:N thread pool?
   - Memory management: Who owns the Future? When is it freed?
   - Performance: Zero-cost async (like Rust) vs runtime overhead (like Go)?
   - Example: `async func:compute = int64() { await delay(1000); pass(42); }`

3. FFI DESIGN FOR C INTEROP (CRITICAL FOR NIKOLA INTEGRATION)
   - Foreign function declaration: `extern "C" func:tesla_valve_init = void();`
   - C struct mapping: How do Aria structs map to C structs?
   - Calling conventions: SysV AMD64 ABI on Linux, Microsoft x64 on Windows
   - Name mangling: How to prevent Aria name mangling for C functions?
   - Header file generation: Can Aria generate .h files for C consumers?
   - Nikola consciousness substrate: C++ library with 9D toroidal wave models
   - Example: `extern "C" func:nikola_process_wave = void(wild float64*:wave_data, size:len);`

4. MEMORY SAFETY ACROSS FFI BOUNDARIES
   - Wild pointers to C: Who owns memory allocated in C?
   - C pointers to Aria: Can C hold pointers to Aria objects?
   - Garbage collector and FFI: Can GC track C-allocated memory?
   - Arena allocators and FFI: Can C functions allocate from Aria arena?
   - 6-stream I/O and C: How to pass FDs 0-5 to C code safely?
   - Lifetime guarantees: How to prevent C from holding dangling pointers?
   - Example: Aria calls C library, C library calls back into Aria with pointer

5. STANDARD LIBRARY ORGANIZATION (MODULES, NAMING CONVENTIONS)
   - Module system: `use std.io;` vs `import "std/io.aria";`
   - Standard library layout: What belongs in std vs third-party packages?
   - Naming conventions: snake_case? camelCase? Type names?
   - Core modules proposal: std.io, std.mem, std.collections, std.async, std.process
   - Minimal stdlib philosophy: Keep it small, push complexity to packages?
   - Versioning: How to handle std lib evolution without breaking code?
   - Example: `use std.collections; vec = vector<int64>.new();`

6. PACKAGE SYSTEM INTEGRATION WITH LANGUAGE FEATURES
   - Package declaration in source: `package:mylib = { ... };`
   - Dependency specification: In code or separate manifest?
   - Visibility modifiers: public/private across package boundaries?
   - Semantic versioning: How to enforce compatibility?
   - AriaX integration: How does package manager find and install dependencies?
   - Compilation units: How to compile packages separately and link?
   - Example: `use pkg.mylib.math; result:x = mylib.sqrt(16);`

7. COMPILATION MODEL (MODULES, IMPORTS, SEPARATE COMPILATION)
   - Module vs file: Is one file one module, or can files contain multiple modules?
   - Import resolution: How to find imported modules? Search paths?
   - Separate compilation: How to compile one file without recompiling dependencies?
   - Interface files: Do we need .ariai (interface) separate from .aria (implementation)?
   - Incremental compilation: How to detect which files need recompilation?
   - Link-time optimization: Can we do cross-module inlining?
   - Build system integration: How does AriaBuild orchestrate compilation?
