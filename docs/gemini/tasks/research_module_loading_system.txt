## PROBLEM STATEMENT

Aria has a **partially implemented module system** but lacks the loader/importer that connects `use` statements to actual stdlib files and imports their symbols into scope.

**Current State:**
- ✅ `UseStmt` parsing works (parser.cpp:1926+)
- ✅ Module syntax supported: `use std.io;`, `use std.io.*;`, `use std.io as io;`
- ✅ Runtime library fully implemented (all C functions exist)
- ✅ Stdlib modules created (`stdlib/io.aria`, `stdlib/core.aria`)
- ❌ **Module loader missing** - compiler doesn't know how to load stdlib/io.aria
- ❌ **Symbol importer missing** - exported symbols from modules not added to scope
- ❌ **Extern linkage incomplete** - `extern func:aria_stdout_write` not linked to runtime

**Test Case Failing:**
```aria
use std.io;

func:main = int64() {
    io.stdout.write("Hello, World!\n");  // Error: Undefined identifier 'io'
    pass(0);
};
```

Error: `Undefined identifier: 'io'` because module loading doesn't happen.

## CONTEXT

### What We Have

**1. UseStmt AST Node** (`include/frontend/ast/stmt.h`)
```cpp
class UseStmt : public Stmt {
public:
    std::vector<std::string> path;  // ["std", "io"]
    std::string alias;               // "io" if aliased
    bool isWildcard;                 // true for std.io.*
    bool isFilePath;                 // true for "file.aria"
};
```

**2. Stdlib Module Structure**
```
stdlib/
  ├── core.aria    (result<T>, array<T>)
  └── io.aria      (6-stream I/O, all extern declarations)
```

**3. Runtime Library** (libaria_runtime.a)
- All C functions exist: `aria_stdout_write()`, `aria_stderr_write()`, etc.
- Headers in `include/runtime/streams.h`

**4. Module Syntax Examples**
```aria
use std.io;              // Import namespace
use std.io.*;            // Import all symbols
use std.io as io;        // Alias
use "custom.aria";       // File path
```

### What We Need

**Module Resolution:**
1. When `use std.io;` is encountered, find `stdlib/io.aria`
2. Parse the module file (full Aria source)
3. Extract exported symbols (`pub func:`, `pub struct:`, etc.)
4. Add symbols to current scope with proper namespace (`io.stdout`, `io.print`, etc.)

**Extern Function Linking:**
1. When `extern func:aria_stdout_write` is declared, mark it for linking
2. During LLVM IR generation, create declaration (not definition)
3. Let linker resolve to `libaria_runtime.a` at link time

**Symbol Table Integration:**
1. Module symbols live in separate namespace
2. `use std.io;` creates `io` namespace with all pub symbols
3. `use std.io.*;` imports symbols into current scope directly
